# CLAUDE.md

Unmelting 협업용 핵심 요약. **현재 코드 기준**으로만 유지한다.
**Unmelting_Game_Concept.md**문서를 필수로 참고한다.

## 프로젝트 요약
- 장르: 3레인 × 3행 카드 레인 로그라이트.
- 플레이: `녹지 않는 소녀` 1장 중심 + 손패/조합/유물.
- 톤: 따뜻한 촛불, 낡은 종이, 어두운 남보라/먹색 다크판타지.

## 실행/검증
```bash
npm install
npm run dev
npm run type-check
npm run test
npm run build
```

## 코드 구조(매니저 맵)
- `src/index.ts`(~3.5k줄): 컴포지션 루트. 턴 루프/손패 해석/점수·활동 로그/보드 채움과 매니저 배선(deps 주입)만 남긴다.
- `src/app/` 런 흐름 매니저: `DevCommandPalette`(디버그 팔레트) · `SettlementScreen`(정산) · `CompanionDirector`(에나 바크/예측/클러치/런 종료 성장 적응) · `RelicEffectsManager`(유물 발동) · `ShopFlowManager`(상점/제단/강제 시련 흐름) · `EventFlowManager`(이벤트 문 흐름) · `FeedbackTypes`(트레일 공용 타입).
- `src/ui/GameBoardRenderer.ts`(~4.5k줄): 렌더 퍼사드. 보드/HUD/손패 렌더와 카운터 롤링을 소유하고, 나머지는 `readonly` 컴포지션 서브뷰로 위임한다. 외부(index/BossEvent)는 퍼사드 공개 API만 호출한다.
- `src/ui/renderer/` 서브뷰: `RendererTypes`(뷰 계약 타입) · `Html`(escapeHtml) · `CardFaceRenderer`(카드/유물 face HTML, `faces`) · `CompendiumView`(도감) · `ExperienceView`(경험 탭) · `ResourceTrailFx`(자원 트레일 엔진, `trails`) · `JobSelectView`(직업 선택) · `EventOverlayView`(이벤트 씬/미니게임) · `BossFxView`(보스 연출/악마 커튼) · `ShopOverlayView`(상점 오버레이/셔터/팩 피커).
- 서브뷰 패턴: `constructor(private readonly host: GameBoardRenderer)` + `import type`(런타임 순환 없음). 렌더 상태 단일 출처는 호스트(공유 접근자 getGameState/getLockedCardIds 등). 새 뷰 분리 시 같은 패턴을 따른다.
- 앱 매니저 패턴: deps 인터페이스 주입. 수치 상태(불빛/화폐/펄스 키)와 게임 플래그는 index가 소유하고 getter/setter 브리지(`resources`)나 클로저로 공유한다. 데이터 테이블/시스템 클래스는 deps가 아니라 직접 import 한다.

## 구현 우선 사실(2026-06-13 기준)
- 거점(촛대) 화면: `/시작` 진입점(`src/ui/hearth/HearthScene.ts`). 인게임 빈 레일 배경 + 오로라 커튼 열림 → 9칸 모두 모험 칸과 같은 어둠→점등 beat로 임시 해방. 모험 클릭 시 `hearth_bg_003` 셔터 배경이 내려온 뒤 흐림/암전되고, 좌측 소개·우측 대형 플레이어 카드·하단 커버플로우 캐릭터 선택이 순차 진입한다. 무역 클릭 시 `hearth_bg_004` 셔터 배경을 암전/블러 없이 그대로 보여 주고, 좌측 2:8 텍스트 전용 탭 라벨(임시 1번~6번)과 우측 중앙 정렬 빈 카드팩 5개 가로 슬라이드 레일이 진입한다. 탭은 hover/active 시 글자 발광·튀어나옴·강한 뒤 그림자로 표현하고, 카드팩은 하단 타원 그림자로 떠 있는 느낌을 준다. 만찬 클릭 시 `hearth_bg_005` 배경 위 중앙 레일에서 무료 만찬 팩 → 음식 1회 + 추가 스탯 2회 3택을 진행하고, 선택 중 화면은 아주 약한 암전/블러로만 배경을 살짝 죽인다. 완성 음식은 실제 유물 `최후의 만찬(last-supper)` 카드로 유물 인벤토리에 깊게 꽂히며, 카드 이름/효과/하단 설명은 선택한 음식·소스·토핑 조합과 각 스탯 합산값으로 생성된다. 스탯 효과는 로비 획득 즉시가 아니라 인게임 시작 시 1회 발동한다. 해당 카드가 인벤토리에 남아 있는 동안 재입장 시 `hearth_006` 일러스트와 고정 대사 화면으로 남는다. 캐릭터 선택은 순환/클릭/드래그와 마지막 본 인덱스 localStorage 복원을 지원하며, 하단 슬롯은 텍스트 없는 직사각 일러스트 마스크로 표시한다. 0번은 튜토리얼 `player_000` 새싹 병아리다. 9칸 배치/역할은 기획서 `Unmelting_Game_Concept.md` §12에 명시. 서고 칸은 항상 개방이며 클릭 시 셔터 하강 후 도서관 톤 배경(`hearth_006` 재사용) 위 상단 '모험일지' 분류 탭과 통산 기록(`LifetimeRecord`) 점선 원장을 보여 준다 — 기록은 `HearthHandlers.getLifetimeRecord`로 주입. 부팅은 검은 타이틀 게이트 뒤 거점 로비로 진입한다: 게임 테마의 네 꼭짓점 반짝 다이아 5개가 실제 로딩(폰트 + 핵심 스프라이트 프리로드, 태스크별 8초 상한)에 따라 점등되는 게이지 → 완료 시 'Click to Start' → 클릭 1회가 오디오 언락·모바일 전체화면을 흡수하고, 장면(로비/런) DOM을 게이트 아래에서 먼저 구성한 뒤 오버레이를 걷는다(커튼 첫 프레임 보전). 최초 실행(`unmelting.boot.firstRunStarted` 미마킹)만 로비 없이 새싹 병아리 직행 — 마킹은 런을 온전히 마친 endGame(승리/패배 정산) 시점에만 하므로, 중도 이탈 후 재부팅은 다시 첫 시작(새싹 직행+인트로)을 탄다 — 이때 첫 실행 인트로 시네마틱이 1회 재생된다: 게이트가 걷혀도 어둠(방사 베일) 유지 → 중앙부터 퍼지는 방사 밝힘으로 배경 노출(상단 구역 커튼·배경 크로스페이드 생략, 완성된 배경을 즉시 깔아 둠) → 플레이어 카드가 하스스톤 하수인 놓듯 중앙에 후웅 안착(+버스트) → 중앙에서 오프닝 대사('역경 아래…') 완료 후 하단 정위치로 하강 → 레일 하강+좌우 확장 → 온보딩 칸 행별 드롭 → 좌우 패널/불씨 HUD 슬라이드 인(오프닝 대사는 이 경로에서 재발화하지 않음). 레일 드롭은 rail-in(1회 재생)→rail-done(정적) 클래스 전환으로 재렌더 시 반복 재생을 막는다. 부팅 게이지 별은 실제 로딩 완료 비율을 따르되 별당 최소 150ms 페이싱으로 순차 점등한다(1구역 배경·새싹 난이도 아트 포함 프리로드). 도감/경험 오버레이는 열림 시 배경 페이드+본문 상승 슬라이드로 등장한다. `/테스트` 명령·`?test=1` 부팅은 메타 전부 해금 + 테스트 런 직행(메타 게이팅 우회). 난이도 캐러셀 포커스는 마지막 선택 저장값을 그대로 따른다(저장 없음 + 쉬움 개방 시에만 쉬움 기본). 모험 셔터의 캐릭터 선택은 에나(0번) 고정으로 패스되어 곧바로 난이도 선택으로 진입한다(추가 동행 해금 전까지). 거점 칸은 모험만 상시 개방, 무역은 새싹 졸업 시 개방(서고 포함 나머지는 잠금·더 어두운 톤, 다음 목표인 무역 잠금 칸만 살짝 밝고 자물쇠가 맥동). 새싹 졸업은 화폐(currency) 메타도 함께 해금하며, 졸업 후 첫 로비 도착에서 전 칸 점등이 끝난 뒤 무역 칸 개방 축하(발광→자물쇠 흔들림→팡! 버스트)와 같은 beat에 화폐 패널이 등장한다(`unmelting.hearth.tradeUnlockCelebrated`로 1회 보장). 새싹 병아리 런에서는 화폐($) 획득 요소를 전부 잠근다: 동전 손패(온보딩 밴 목록), 무료 카드 1$ 결과, 자원팩 동전 한 닢, 마리골드(코인 꽃) 스폰 제외 — 졸업 후 정규 런부터 전부 열린다. `/리셋` 명령은 전체 진행을 초기화한다: 에나 3종 키(성향 `ENA_DISPOSITION_STORAGE_KEY`·자기학습 `ENA_SELF_LEARNING_STORAGE_KEY`(유대·모험 xp·특화·기억·first-seen)·정책망 `ENA_POLICY_STORAGE_KEY`)에 더해 `unmelting.` 접두사 localStorage 전부(통산 기록·메타 해금·거점 마지막 캐릭터/난이도·개발 개방)를 지운 뒤 재부팅한다 — 새 진행도 키는 `unmelting.` 접두사만 지키면 자동으로 리셋 대상이 된다.
- 정산(온보딩 클리어/100F 클리어/사망) 화면은 `app/SettlementScreen` 단일 매니저가 같은 골격으로 렌더하고, 우측 하단(모바일은 카드 아래 중앙)에 통산 기록 두 줄을 표기한다. `/사망`·`/승리` 디버그 명령으로 실제 종료 흐름 그대로 즉시 점검할 수 있다.
- 모바일(≤700px) 보정: 거점 칸 자물쇠/라벨 축소(잠금 아이콘은 pointer-events 차단 + 칸 overflow hidden으로 이웃 칸 클릭 가로채기 방지), 셔터 뒤로가기 버튼은 셔터 바로 아래로 이동, 서고 원장·정산 화면 세로 플로우 축소판, 말풍선은 `width: max-content`+최종 문장 선측정으로 화면 가장자리 클램프(세로 한 글자 접힘 방지). 터치 기기 거점 칸은 첫 탭 인스펙터·둘째 탭 진입.
- 턴 루프: 플레이어 행동 → 적/보물/폭탄/꽃 처리 → 불씨/레일 하강·리필 → 포자 카운트 0 표시 → 전염/2턴 리셋. 굳음 해제도 0턴을 짧게 표시한 뒤 해동 연출로 풀린다.
- 보스도 밀랍 굳음 적용 대상이다. 굳은 보스는 직접 공격 피해와 행동 비용(불씨 감쇠 등)은 받지만 보스 반격 주기/공격 카운트다운만 줄지 않고 반격·특수행동을 건너뛰며, 굳음 UI는 보스 전용 face 중앙 텍스트(상단 굳음, 하단 n턴)를 사용한다.
- 상점 루프: 10/20턴 상점, 30턴 제단 모드.
- 상점/제단 콘텐츠는 `shop-content-bundle` 레이어로 **동시 공개**.
- 시작 직업 선택은 빈 3×3 레일 위 레일 내부 암막/커튼 레이어에서 진행하고, 선택 후 최초 보드를 채운 뒤 커튼을 걷는다.
- 이벤트 문 진입은 레일 커튼이 충분히 닫힌 뒤 문 소비 → 짧은 딜레이 → 씬 일러스트 세로 슬릿(가로 확장+동시 페이드) 공개 → 대사 → 선택 버튼 순서이며, 종료는 버튼 숨김 → 마무리 대사 → 일러스트 완전 접힘 → 커튼 열림 뒤 입력을 해제한다.
- 기방문 스킵: 한번 본 이벤트(`unmelting.seen.event.<id>`)/보스(`unmelting.seen.boss.<kind>`)는 레일 우하단 `SKIP >>` 글자 버튼이 떠서 연출·대사만 건너뛰고 본편(선택지/미니게임/전투)으로 바로 들어간다(효과·정산은 그대로). 이벤트는 마무리 대사까지 생략, 보스는 인트로 대사만 생략(등장 애니/타이틀 유지). 새싹 병아리(온보딩)는 무역 1번 탭 `이벤트` 개방(`MetaUnlockId 'events'`) 전까지 이벤트 문이 스폰되지 않는다.
- 미니게임 에나 대사: `EVENT_MINIGAME_LINES`(minion-good/jackpot/sting, rps-win/tame/lose/streak/relic) 풀을 렌더러 `onMoment` 훅이 결과 확정 순간에 알리고 index가 발화한다 — 잭팟/유물/연승은 확정, 잔잔한 승패는 0.5 확률. 대사 길이는 `CompanionLines.test.ts`가 렌더 결과 56자 상한으로 가드한다(너무 긴 대사는 읽히지 않는다).
- 제단: 리롤 없음, 무료 카드 2장(기본+동전 한 닢 1$), 팩 4종(축복/자원/강화/삭제). 동전 한 닢은 제단 방문마다 초기화한다.
- 제단 자원팩은 8종 풀(두꺼운 의복/가열/큰 배낭/성냥갑/두꺼운 심지/조커 카드/랜턴/동전 한 닢)에서 weight 기반 3장을 제시한다.
- 상점: 유물 슬롯 리롤(무료/팩 고정)과 팩 피커 재뽑기(1+횟수$ 화폐)를 별도로 지원한다. 팩 가격은 방문 내 동일 팩 구매마다 시작가를 더한다. 일반 상점 3팩(자원/조합/해금) 공통 시작가는 10층 120에서 10층마다 +40(20F 160, 40F 240 …), 제단은 30F 500·60F 1500·90F 2500.
- 무료 카드(타이틀 상점/제단 공통 `무료 카드`): 방문마다 등록된 n종 중 랜덤 효과 1개, 즉시 소모.
- 손패 `동전`은 보물상자 전용 보너스 드롭이며 일반 적 처치/드로우 풀에는 섞지 않는다.
- 키틴은 전방 선택 함정 1장을 제거한다. 단일은 2칸 이하, 트리플은 3칸 이하 폭(groupCount)까지 허용한다.
- 보스 현상금은 1~3$이며 1$ 단위 트레일/카운터로 증가하고, 전리품 유물 보상 트레일 목적지는 보유 유물 팬이다.

- Shift 자세히보기: 생성자에서 `is-shift-detail` 초기 클리어 후 keydown/keyup·window.blur·boardElement mousemove(e.shiftKey 확인) 로 클래스 잔류를 방지. 도감 chip은 `display:none↔inline-block`으로 전환(phantom-width 방지); 손패 미리보기는 opacity 0.18s 페이드. `.codex-chip-body`로 label+value를 단일 flex item으로 묶어 두 칸 분리 레이아웃 방지.
- 손패 hover 미리보기는 읽기용 고정 표면이다. 같은 손패 UID 위에 커서가 머무는 동안 레일 하강/HUD/효과 리렌더가 발생해도 기존 미리보기 DOM을 되심고 stable 클래스로 앞면 열린 상태를 고정해 flip·카드 뒷면 애니메이션을 재실행하지 않는다.
- 사치품 유물 최대 공격력 보너스 +3. 붉은 포션은 직접 공격·손패·체인 포함 모든 처치에 적용된다.
- 샹들리에는 기본 해금(runLocked: false). 자원팩 갑옷 장착 방패 +5. 시련 '역경' 적용 시 기존 함정 카드에도 즉시 피해 보너스 반영.
- 유물/팩/무료카드 희귀도는 `ShopPools` 공통 팔레트(common~legendary) 사용. 상점/제단 유물은 등급별 발광을 유지한다.
- 에나 지원 카드 판단은 `src/systems/HandCardAdvisor.ts` 범용 스코어러가 "기대 HP 절약/자원 가치" 환산으로 계산한다(청소=임박 병합 실효 피해×확률·3칸 즉사 후보는 최상위 고정·1칸 웹 `WEB_FIELD_CLUTTER_THRESHOLD`장 이상이면 병합각 없어도 광역 청소, 처치=실효 공격력×반격 임박도, 방어=흡수 기대 피해, 성냥=불씨 소멸 스폰 압박 비용, 회복=저체력 비선형; 고정 서열은 동점 ε 타이브레이크로만). 레일 예고 큐(같은 위협 추가 도착)·트리플 완성각(비합체 2장 보유 지급 허용+가점)·`EnaDisposition.supportRoleWeights`(청소/처치/방어/자원/회복 피팅 가중, 기본 1.0) 보정을 받는다. 후보가 보유 손패의 마지막 재료가 되어 해금 레시피(`unlockedRecipeIds` 스냅샷)를 완성하면 효과를 같은 축(불씨/회복/피해/청소)으로 보수 매핑해 `RECIPE_SUPPORT_DISCOUNT`(0.55) 할인 가산한다 — 직접 효과 카드가 항상 레시피 경로보다 우선하고, 필요와 안 맞는 완성각은 단독 추천 근거가 못 된다. 새 손패/유물은 데이터 테이블(HandCards/Relics)에 `synergyTags`(+공격 카드는 `damageProfile`)를 달아 등록하면 에나 판단(HandCardAdvisor)·지식(EnaKnowledgeAdapter)·시뮬 학습에 자동 반영된다 — 에나 지원 카드를 코드에 하드코딩하지 말 것. `damageProfile`은 HandSystem 실제 공식의 보수 근사이므로 공식 변경 시 함께 갱신한다.

## 불씨 티어별 스폰 압박

- 티어는 절대 ember 수치 기준(bright ≥7, dim ≥4, flickering ≥1, extinguished =0)이며 emberMax와 무관하다.
- 티어 하강 시 **적·함정 가중치를 높이는** 방식으로 압박을 표현한다. 보물은 extinguished에서도 최소 5 유지.
- `CardSpawner`에서 `Math.max(1, ...)` 하한 보장으로 보물이 0이 되는 상황을 추가 방지한다.

- 제단 EXIT 이후 보스 임시 플로우를 확장해 (강하 인트로→타이틀 슬라이드→보스 전용 턴 루프/3턴 공격→보상 3칸x3 강제 획득→강제 시련 3택) 순서를 고정했다.

- 제단 보스 인트로는 3x3 보스 칸 낙하 후 진입하며, 보스 전투는 플레이어 수동 공격(기본 공격만 턴 소비) 기준 3턴 주기 피격으로 처리한다.
- 보스 보상은 3칸 보상 3종을 레일형 드롭 선택으로 모두 획득해야 진행되며, 이후 시련은 상점형 레이어+3분할 카드 선택 후 셔터 상승으로 일반 턴에 복귀한다.

- 보스전은 별도 게임 화면이 아니라 일반 레일 톤 이벤트로 유지하며, 거대 1칸 보스 타일(3턴 반격)→레일형 3보상 드롭→상점톤 시련 탭(EXIT 후 셔터 상승) 순서를 고정했다.

- 보스 이벤트의 1/2/3턴은 가상 턴이며 실제 런 턴 카운터(예: 30턴)는 전투/시련 종료까지 고정한다.

- 강제 시련은 독립 팝업 대신 상점과 같은 shell/content-bundle 레이어를 재사용해 선택→EXIT→셔터 상승 순서를 동일하게 유지한다.

- 강제 시련 카드는 데이터 테이블(FORCED_TRIAL_CARDS)로 관리하며, 선택 시 즉시 런 modifier를 적용하고 EXIT는 선택 이후에만 허용한다.

- 60F 보스 `불씨 기사단장`은 숨은 정체가 레온하르트이며, 플레이어 응답 대사만 이를 암시한다. 3×3 구조, boss_002, 좌→우 느린 진입 후 쿵 착지, 저택의 방패 타이틀, 3턴마다 기사단장의 손패 2장을 사용한다. 방패 획득 시 플레이어처럼 HP 게이지 좌상단의 큰 방패 아이콘 안에 수치를 표시한다.

- 에나 성장 곡선: 신규 플레이어는 ROOKIE_DISPOSITION(개입을 BASE의 ~55%로 낮추되 대사는 BASE 동일. 위기 배율 adversityBoost는 억제 없이 1.2로 실효화해 '극단적일수록 화려한 도움'이 성립)에서 시작하고, `computeEnaGrowth({adventureXp,bond})`(EnaDisposition.ts, 1-exp, 메타 상점 해금형 전환 시 이 함수만 교체) 성장值가 평균회귀 앵커를 ROOKIE→BASE로 옮긴다. 모험 xp는 런 횟수가 아니라 4축(런 완료 소량+층 등반 주축+의사결정+진행 턴, 축별 캡, `ENA_RUN_XP_TUNING`) 합산으로 자살런은 극소량이며, 일반 런은 매판 소폭(얕은 런 1~2%p) 상승·첫 경험(보스 첫 격파/희귀 유물/제단/별빛/카드 해금, 1회성 `experienced` 저장) 소량 가산·희귀 점프(5~10%p)는 자격(기록 대폭 경신/보스 층)+드라마 점수(위기감/도움 실효/고전/새 시도 4계열 캡 합산, `ENA_RUN_DRAMA_TUNING.jumpThreshold`) 게이트를 함께 넘어야 한다. 적립은 `EnaAutonomousLearner.accrueAdventureXp`(구 저장본은 totalRuns×`LEGACY_XP_PER_RUN` 1회 이전), 초보일수록 미숙 대사 확률 가점(0.45→0.70). 비긴급 에나 바크는 `BarkSequencer`(ui) 순차 큐(최소 노출 1.5~3초, 상한 3, 낮은 중요도부터 드롭)로 출력되고 urgent/클러치는 즉시 교체를 유지한다.
- 에나 축 특화 초과 성장: 개입 4축(예지 predict/수호 protection/온정 minor/불굴 grit) 특화 점수(0~1, 단조·하락 없음)가 자기학습 저장(version 1, optional `specialization`)에 영속되고, 런 종료 시 제때 예측·클러치 목격·소소한 클러치/보물·(받은 피해×깊이) 신호로만 아주 소량 적립된다(축별/런당 총 캡·얕은 런 감쇠·잔여분 비례 포화는 `ENA_SPECIALIZATION_TUNING`). `clampDisposition(d, spec)`이 특화 축 노브 상한만 `hi×(1+spec×SPECIALIZATION_MAX_EXTENSION)`(최대 2배)로 확장하고 평균회귀 앵커도 `specializedAnchorDisposition`(리프트 0.35)으로 확장 상한 쪽으로 상향돼 평형점이 실제로 올라간다 — 특화 0이면 기존 동작과 완전히 동일하며, 경험 탭 표시는 실질 기반이라 자동 추종하고 clamp01 100% 상한만 유지한다(grit 특화 1 앵커 ≈ 불굴 표시 ~64%, 90%대는 확장 상한까지 자란 값에서만). 저장/로드(`saveDisposition`/`loadDisposition`)에도 특화를 함께 넘겨 초과값이 잘리지 않게 한다.
