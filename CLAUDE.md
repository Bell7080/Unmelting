# CLAUDE.md

Unmelting 작업용 최소 컨텍스트. 오래된 아이디어보다 **현재 구현**을 우선한다.

## 프로젝트 한 줄

- 3레인 × 3행 필드에서 카드가 매 턴 전진하고, 플레이어는 `녹지 않는 소녀` 한 장과 손패/조합/유물로 위험과 보상을 처리하는 카드 레인 로그라이트.
- 톤: 따뜻한 촛불 + 낡은 종이 + 어두운 남보라/먹색의 몽글몽글 다크판타지.

## 실행/검증

```bash
npm install
npm run dev
npm run type-check
npm run test
npm run build
```

- 배포 타깃은 Vite `dist`.
- 시각 변경이 크면 브라우저 확인/스크린샷을 남긴다.

## 현재 구조

```text
src/index.ts                 # 런타임 조립, 턴 흐름, 불빛/화폐/상점, 이벤트 핸들링
src/core/GameState.ts         # 중앙 상태, 레인 압축/리필, 턴 카운트
src/core/TurnManager.ts       # 적/폭탄/보물 변이/불씨 턴 처리
src/entities/                # Card, Character, Lane, HandCard 모델
src/systems/                 # Action, CardSpawner, Drop, Ember, Hand 시스템
src/data/                    # 손패/유물/조합 정의
src/ui/GameBoardRenderer.ts  # DOM 렌더링, 상호작용, 대부분의 UI 애니메이션
src/ui/styles/              # GameBoardRenderer 주입 CSS를 역할별 TS 문자열로 분리
src/ui/SquareBurst.ts        # 사각 파티클 버스트
src/ui/Icons.ts              # 플랫 inline-SVG 아이콘
src/ui/Sprites.ts            # webp 스프라이트 매핑
src/assets/                  # 폰트/스프라이트
```

## 구현된 핵심 시스템

- 필드: 3레인 × 3행. 전방 행만 직접 클릭 처리, 같은 전방 카드 그룹화. 꽃(버프칸)은 씨앗으로 대기 라인에만 생성되고 전방 도착 시 5종 꽃 중 하나로 발화한다.
- 턴: 플레이어 행동 → 이벤트/적/폭탄/보물 변이 → 불씨 감소/리필/그룹화. 낮은 불씨 단계에서는 적 선공.
- 손패: 시작 손패 없음, 드롭, 자동 3장 병합, 타깃형/즉발형 효과, 체인 타임라인, 조합 발동, 촛불 게이지. 새 손패는 게이지 바로 아래에서 획득 순서대로 일정하게 하강하고, 보상 드롭이 모두 착지한 뒤 자동 트리플을 검사한다. 신규 트리플은 병합 직전 원본 카드 id를 저장하고 실제 DOM 복제 레이어 2장이 각자의 이전 위치에서 가장 하단 병합 슬롯으로 합류한 뒤 젤리/사각 버스트로 합성감을 준다. 손패 10장까지는 카드 높이를 먼저 압축하고 겹침은 최대한 피한다.
- 불빛/화폐/수치 트레일: 좌측 패널에 종합 점수 대신 **불빛**, 상점 지갑은 **화폐**로 표시한다. 현재 상점 구매 비용은 불빛을 사용한다. 손패/불빛/화폐/체력/방패/불씨/게이지/공격력 획득은 `index.ts`의 공통 목적지 규칙표를 따라 출처 칸/체인/중앙 사용 카드에서 목적 HUD로 사각 카드 꼬리 블라스트가 꽂힌 뒤 수치/드롭 변화와 SquareBurst가 같은 beat에 재생되어야 한다. 유물 구매는 불빛 패널에서 클릭한 유물 카드로 역방향 지불 트레일을 쏜다.

- 꽃: 시작 3×3에는 등장하지 않는다. `flower_000` 씨앗이 전방에 오면 `flower_001~005` 중 무작위 발화. 캐모마일=불빛, 레드로즈=회복, 메리골드=화폐(2턴마다 성장), 올레안더=방패, 라벤더=손패 게이지. 꽃은 합쳐지지 않고, 성장할수록 10%부터 급상승하는 시듦 확률을 굴려 `enemyflower_001` 괴물꽃(꽃 수확값만큼 HP/ATK)으로 변한다. 괴물꽃은 서로끼리만 합쳐진다.
- 상점/유물: 10턴마다 레일 셔터 후 닫힌 레일 위 유물 상점. 셔터는 전방 2/3칸 그룹을 한 패널로 덮고, 가격은 플랫 태그 라벨로 표시한다. 셔터가 내려가면 체인은 즉시 초기화하고, 셔터가 닫힌 동안 폭탄 점화 글로우 같은 레일 반복 이펙트만 일시정지한다. 셔터 잠금 상태는 렌더러 상태로 보존해 유물 클릭/구매로 보드가 재렌더되어도 사라지지 않는다. 유물 카드 하강은 첫 오픈에만 재생하고, 구매 갱신은 DOM을 제자리 갱신해 무애니메이션으로 처리한다. EXIT 후 카드 퇴장 완료 → UI 정리 → 레일 덜컹임 → 셔터 역방향 상승 순서를 지킨다.
- 도감: 적/함정/보물/꽃/손패/조합/유물/용어 탭. 실제 데이터 정의를 요약 표시한다. 성수는 기본 사용 시 필드 랜덤 포자 2장 제거, 트리플 사용 시 필드 전체 포자 제거다.

## UI/UX 규칙

- 기존 촛불/밀랍/낡은 카드 테마를 유지한다. 새 UI는 `GameBoardRenderer.ts`의 색/테두리/그림자/스크롤바 양식을 먼저 참고한다.
- 폰트는 `FontManager`와 OkDanDan을 사용하고 최소 12px 원칙을 깨지 않는다.
- 이모지 아이콘을 새로 넣지 않는다. 도감/검/하트/코인/방패/상점 보석처럼 **`src/ui/Icons.ts`의 플랫 inline-SVG path 아이콘**을 사용하거나 같은 방식(`currentColor`, 단색 fill/stroke, 작은 크기 가독성)으로 추가한다. 플레이어 카드의 방패 칩은 도감 런처와 같은 따뜻한 금색 계열을 따른다.
- 불빛/화폐/손패/체력/방패/불씨/게이지/피해/회복 등 즉각 피드백은 출처 블라스트, 목적지 버스트, 수치/드롭 변화가 분리되어 보이지 않게 같은 beat 안에서 처리한다. 꽃 관련 버스트는 공통 `flower-bloom`만 쓰지 말고 꽃 종류별 팔레트를 우선한다.

## 코드 작성 규칙

- TypeScript만 사용한다. import 주위에 try/catch를 두지 않는다.
- 새 코드에는 다음 작업자가 의도를 알 수 있는 짧은 주석을 남긴다. 단, 자명한 문법 설명은 피한다.
- 상태 변경은 가능한 한 시스템/엔티티에 두고, 렌더러는 표시/애니메이션 책임을 유지한다.
- 테스트용 시드, 더미 UI, 완료된 과거 실험 코드는 남기지 않는다. 발견 시 제거하거나 최종 보고에 명시한다.
