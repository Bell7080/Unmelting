# CLAUDE.md

Unmelting 협업용 핵심 요약. **현재 코드 기준**으로만 유지한다.
**Unmelting_Game_Concept.md**문서를 필수로 참고한다.

## 프로젝트 요약
- 장르: 3레인 × 3행 카드 레인 로그라이트.
- 플레이: `녹지 않는 소녀` 1장 중심 + 손패/조합/유물.
- 톤: 따뜻한 촛불, 낡은 종이, 어두운 남보라/먹색 다크판타지.

## 실행/검증
```bash
npm install
npm run dev
npm run type-check
npm run test
npm run build
```

## 구현 우선 사실(2026-05-21 기준)
- 턴 루프: 플레이어 행동 → 적/보물/폭탄/꽃 처리 → 불씨/레일 하강·리필 → 포자 카운트 0 표시 → 전염/2턴 리셋.
- 보스는 디버프(굳음) 면역: 밀랍 시도 시 즉시 저항 연출 후 굳음 스택을 남기지 않는다.
- 상점 루프: 10/20턴 상점, 30턴 제단 모드.
- 상점/제단 콘텐츠는 `shop-content-bundle` 레이어로 **동시 공개**.
- 제단: 리롤 없음, 무료 카드 2장(기본+수당 3$), 팩 4종(축복/자원/강화/삭제). 수당은 제단 방문마다 초기화한다.
- 제단 자원팩은 8종 풀(두꺼운 의복/가열/큰 배낭/성냥갑/두꺼운 심지/조커 카드/랜턴/동전 한 닢)에서 weight 기반 3장을 제시한다.
- 상점: 리롤은 유물 슬롯만 교체(무료/팩 고정)하며 정산 중 재클릭을 잠근다. 팩 가격은 방문 내 동일 팩 구매마다 시작가를 더한다(일반 120/500/520, 제단 30F 500·60F 1500·90F 2500).
- 무료 카드 `선물 상자`: 방문마다 등록된 n종 중 랜덤 효과 1개, 즉시 소모.
- 손패 `동전`은 보물상자 전용 보너스 드롭이며 일반 적 처치/드로우 풀에는 섞지 않는다.
- 키틴은 전방 선택 함정 1장을 제거한다. 일반은 2칸 이하, 트리플은 3칸 이하 폭(groupCount)까지 허용한다.
- 보스 현상금은 1~5$이며 1$ 단위 트레일/카운터로 증가하고, 전리품 유물 보상 트레일 목적지는 보유 유물 팬이다.
- 90F 조각사 소환 적은 사망 즉시 모델에서 제거한 뒤 보상 손패를 렌더해 재등장 잔상을 막는다.
- 보스/90F 규칙 타이틀은 제목 부상 완료 후 짧은 딜레이가 지나야 하단 클릭 문구가 점멸하고 입력이 열린다.
- 90F 보스 보상과 강제 시련이 끝나 셔터가 오른 직후, 남청~보라빛 `잿빛 굴레` 타이틀 연출과 함께 90~100층 별빛 규칙이 발동된다: `turnkey_001` 비합체 특수 보물 `별빛` 획득만 턴 +1, 다른 행동은 턴을 올리지 않는다.
- 별빛으로 100턴에 도달하면 즉시 클리어하지 않고 100F 보스 `녹지 않는 마녀`로 진입한다. 3×3 boss_004, 2턴 주기, 210/140/70 HP 페이지 경계 초과 피해 컷, HP바 경계선 표시, 1P 손패 소각·2P 4장 동시 전개 보스 손패(중복 시 추가 카드)·3P HP+10/ATK+3 강화 소환을 사용한다. 해금된 페이지 능력은 이후 페이지에도 유지되며 페이지 전환 이벤트는 최초 하향 돌파 1회만 발생한다.
- 불빛 턴 보정은 완만한 선형식(1 + 턴×0.015)으로 유지해 90턴대 경제 폭증을 제한한다.
- 유물/팩/무료카드 희귀도는 `ShopPools` 공통 팔레트(common~legendary) 사용. 상점/제단 유물은 등급별 발광을 유지한다.

## UI/UX 규칙
- 이모지 신규 추가 금지. `src/ui/Icons.ts`의 플랫 SVG 스타일 유지.
- 촛불/밀랍/낡은 종이 테마 유지(색·테두리·그림자 일관성).
- 수치 피드백(불빛/화폐/체력/방패/불씨/게이지)은 트레일/버스트/롤링 카운트가 같은 beat에 보이게 유지.

## 코드 규칙
- TypeScript only, import 주변 try/catch 금지.
- 새 코드에는 의도 중심의 짧은 주석 포함.
- 테스트/더미/과거 실험 잔여 코드는 남기지 말고 제거 또는 보고.

## 문서 규칙
- 장문 패치노트 누적 금지.
- 사실/규칙/주의점만 유지하고 중복 설명은 제거.

- 기존 촛불/밀랍/낡은 카드 테마를 유지한다. 새 UI는 `GameBoardRenderer.ts`의 색/테두리/그림자/스크롤바 양식을 먼저 참고한다.
- 폰트는 `FontManager`와 OkDanDan을 사용하고 최소 12px 원칙을 깨지 않는다.
- 이모지 아이콘을 새로 넣지 않는다. 도감/검/하트/코인/방패/상점 보석처럼 **`src/ui/Icons.ts`의 플랫 inline-SVG path 아이콘**을 사용하거나 같은 방식(`currentColor`, 단색 fill/stroke, 작은 크기 가독성)으로 추가한다. 플레이어 카드의 방패 칩은 도감 런처와 같은 따뜻한 금색 계열을 따른다.
- 불빛/화폐/손패/체력/방패/불씨/게이지/피해/회복/처치 등 즉각 피드백은 출처 블라스트, 목적지 버스트, 수치/드롭 변화, 사망 축소가 분리되어 보이지 않게 같은 beat 안에서 처리한다. 꽃 관련 버스트는 공통 `flower-bloom`만 쓰지 말고 꽃 종류별 팔레트를 우선한다.

## 코드 작성 규칙

- TypeScript만 사용한다. import 주위에 try/catch를 두지 않는다.
- 새 코드에는 다음 작업자가 의도를 알 수 있는 짧은 주석을 남긴다. 단, 자명한 문법 설명은 피한다.
- 상태 변경은 가능한 한 시스템/엔티티에 두고, 렌더러는 표시/애니메이션 책임을 유지한다.
- 테스트용 시드, 더미 UI, 완료된 과거 실험 코드는 남기지 않는다. 발견 시 제거하거나 최종 보고에 명시한다.
