# CLAUDE.md

Unmelting 작업용 최소 컨텍스트. 오래된 아이디어보다 **현재 구현**을 우선한다.

## 프로젝트 한 줄

- 3레인 × 3행 필드에서 카드가 매 턴 전진하고, 플레이어는 `녹지 않는 소녀` 한 장과 손패/조합/유물로 위험과 보상을 처리하는 카드 레인 로그라이트.
- 톤: 따뜻한 촛불 + 낡은 종이 + 어두운 남보라/먹색의 몽글몽글 다크판타지.

## 실행/검증

```bash
npm install
npm run dev
npm run type-check
npm run test
npm run build
```

- 배포 타깃은 Vite `dist`.
- 시각 변경이 크면 브라우저 확인/스크린샷을 남긴다.

## 현재 구조

```text
src/index.ts                 # 런타임 조립, 턴 흐름, 불빛/화폐/상점, 이벤트 핸들링
src/core/GameState.ts         # 중앙 상태, 레인 압축/리필, 턴 카운트
src/core/TurnManager.ts       # 적/폭탄/보물 변이/불씨 턴 처리
src/entities/                # Card, Character, Lane, HandCard 모델
src/systems/                 # Action, CardSpawner, Drop, Ember, Hand 시스템
src/data/                    # 손패/유물/조합 정의
src/ui/GameBoardRenderer.ts  # DOM 렌더링, 상호작용, 대부분의 UI 애니메이션
src/ui/styles/              # GameBoardRenderer 주입 CSS를 역할별 TS 문자열로 분리

src/ui/Icons.ts              # 플랫 inline-SVG 아이콘

src/assets/                  # 폰트/스프라이트
```

## 구현된 핵심 시스템

- 보스/시련 토대: `TurnManager`에 `normal_turn | boss_phase` 모드 분리를 추가해 보스 단계는 일반 턴 카운트에서 제외할 수 있는 코드 경계를 마련했다. 시련은 `trial-overlay` 3선택 독립 UI로 표시되고, 선택 결과는 런 영속 modifier(스폰 가중치/적 스탯/함정 피해)에 누적된다.
- 카드 풀: 메타 사당(추후 구현) 해금 목록과 런 내부 카드풀(`RunCardPool`: unlocked/locked/banned)을 분리하는 이중 해금 구조를 도입했다.
- 디버그 훅(정식 출시 전 제거 예정): `/`로 상단 명령어 팔레트를 열고 `/1~100turn`으로 턴 점프, `/희망` 같은 이름 명령으로 유물/손패를 즉시 지급할 수 있다.
- 필드: 3레인 × 3행. 전방 행만 직접 클릭 처리, 같은 전방 카드 그룹화. 꽃(버프칸)은 씨앗으로 대기 라인에만 생성되고 전방 도착 시 5종 꽃 중 하나로 발화한다.
- 턴: 플레이어 보드 행동 → 적/보물/폭탄/꽃 이벤트 처리 → 불씨 감소와 레일 하강/리필/그룹화 → 포자 전염. 10턴 상점은 이 턴 처리가 끝난 뒤 열리며, 상점 구매/EXIT 자체는 턴·포자 타이머를 진행하지 않는다. 포자 전염 뒤 새로 맞닿은 전방 포자는 즉시 한 번 더 그룹화한다. 보드 행동으로 생긴 전방 빈칸은 적 턴 전에는 내려오지 않아 다음 적이 같은 턴에 때리지 않는다. 손패/조합 효과 중에는 입력 재개 전 보드 준비용 하강/리필을 수행한다. 낮은 불씨 단계에서는 적 선공. 굳음 상태의 폭탄/포자/꽃/보물 같은 턴 타이머 요소는 해동 전까지 타이머가 진행되지 않는다.
- 손패: 시작 손패 없음, 드롭, 자동 3장 병합, 타깃형/즉발형 효과, 체인 타임라인, 조합 발동, 촛불 게이지. 새 손패는 손패 콤보 게이지 바로 아래에서 살짝 내려온 최상단 지점부터 보이는 상태로 획득 순서대로 하강하고, 보상 블라스트는 같은 최상단 지점을 향하며 착지 후 자동 트리플을 검사한다. 신규 트리플은 병합 직전 원본 카드 id를 저장하고 실제 DOM 복제 레이어 2장이 각자의 이전 위치에서 가장 하단 병합 슬롯으로 합류한 뒤 젤리/사각 버스트로 합성감을 준다. 손패 10장까지는 카드 높이를 먼저 압축하고 겹침은 최대한 피한다. 손패 사용뿐 아니라 꽃/유물 등 외부 보상이 촛불 게이지를 10 이상으로 채워도 같은 전체 게이지 보상이 즉시 해소된다.
- 불빛/화폐/수치 트레일: 좌측 패널에 종합 점수 대신 **불빛**, 상점 지갑은 **화폐**로 표시한다. 현재 상점 구매 비용은 불빛을 사용한다. 손패/불빛/화폐/체력/방패/불씨/게이지/공격력 획득은 `index.ts`의 공통 목적지 규칙표를 따라 출처 칸/체인/중앙 사용 카드에서 목적 HUD로 사각 카드 꼬리 블라스트가 꽂히는 순간 수치/드롭 변화와 SquareBurst가 같은 beat에 재생되어야 하며, 서로 독립적인 획득 계산은 순차 대기하지 않고 병렬로 묶는다. 불빛/화폐/체력/방패/불씨/손패 게이지/공격력 숫자와 HP/불씨/게이지 바·틱은 증감 시 이전 표시값에서 목표값까지 정수 롤링으로 띠리리릭 변화하되, 큰 수치 변화는 더 빠른 가속 롤로 처리한다. 불빛/화폐 숫자도 트레일 착지 훅에서 바로 롤링을 시작해 카드 소멸/정리 애니메이션을 기다리지 않는다. 손패 사용이 필드 칸을 피해/파괴/획득/굳음으로 바꾸면 중앙 사용 카드 위치에서 해당 칸으로 타깃 블라스트도 함께 쏴 피격 대상을 먼저 읽게 한다. 유물 구매는 불빛 패널에서 클릭한 유물 카드로 역방향 지불 트레일을 쏜다.

- 꽃: 시작 3×3에는 등장하지 않는다. `flower_000` 씨앗이 전방에 오면 `flower_001~005` 중 무작위 발화. 캐모마일=불빛, 레드로즈=회복, 메리골드=화폐(2턴마다 성장, 중간 턴에는 다음 성장 예고), 올레안더=방패, 라벤더=손패 게이지. 꽃은 합쳐지지 않고, 성장할수록 10%부터 급상승하는 시듦 확률을 굴려 `enemyflower_001` 괴물꽃(꽃 수확값만큼 HP/ATK)으로 변한다. 괴물꽃은 서로끼리만 합쳐진다.
- 상점/유물: 10턴마다 레일 셔터 후 닫힌 레일 위 유물 상점. 셔터는 전방 2/3칸 그룹을 한 패널로 덮고, 가격은 플랫 태그 라벨로 표시한다.
- 상점 셔터 정렬: 셔터 패널은 고정 3×3 비율이 아니라 **실제 렌더된 rail cell의 픽셀 위치/크기**(전방 2/3칸 그룹 포함)를 따라 붙어야 한다. 전/중/상단의 원근 스케일 차이를 그대로 반영해 레이어 미스매치를 막는다.
- 상점: 10턴 셔터 상점은 상/하단 분할형(유물+새로고침, 무료 카드+카드팩+자원 보충) UI를 사용한다. 새로고침/카드팩은 같은 상점 내 구매 횟수만큼 가격이 오른다. 유물은 구매한 만큼 슬롯이 줄어든 상태로 새로고침된다.
- 상점/카드팩/무료카드 표시 등급은 공통 희귀도 팔레트(common/rare/epic/unique/legendary)로 통일하고, 유니크는 예외 메커니즘(예: 희망) 전용으로만 사용한다.
- 상점 카드팩/무료카드/팩 선택지 풀은 `src/data/ShopPools.ts` 데이터 테이블에서 관리한다.
- 상점 루프: 10/20턴은 상점, 30턴은 제단(강화 상점 변형)으로 순환하는 100턴 러닝 구조를 준비 중이다. 보스/시련은 현재 토스트/상태 토대로 연결돼 있으며 실제 전투 규칙은 후속 구현 단계다.
 셔터가 내려가면 체인은 즉시 초기화하고, 셔터가 닫힌 동안 폭탄 점화 글로우 같은 레일 반복 이펙트만 일시정지한다. 셔터 잠금 상태는 렌더러 상태로 보존해 유물 클릭/구매로 보드가 재렌더되어도 사라지지 않는다. 유물 구매 시 상점 카드가 플레이어 오른쪽의 보유 유물 부채꼴 스택으로 날아가 착지한다. 보유 유물 카드는 상점 유물 카드와 같은 그림/이름/효과/플레이버 구조를 가격표 없이 사용하고, 기본 크기에서 읽히며 hover/focus/클릭 고정은 크게 확대하기보다 레이어를 위로 올리는 정도로만 사용한다. 희망 부활은 유물 카드가 손패 사용 미리보기와 같은 중앙 지점으로 이동(전체 보드 줌 없음, hover 확대 비활성)→흔들림→백색 폭발→중앙에서 모든 필드 카드로 블라스트→카드 파괴→체력 블라스트 회복→레일 정리/리필 순서로 연출하고, 부활/대규모 필드 제거 뒤 새 3×3은 전방 포함 전부 상단에서 낙하 리필로 복구한다. 이때 첫 리필 후보는 전방 인접 중복을 피하도록 분리 생성하고, 그래도 겹친 전방 카드는 첫 입력 전 즉시 그룹화한다. 보유 유물이 없으면 오른쪽 유물 레이어 자체를 렌더하지 않는다. EXIT 후 카드 퇴장 완료 → UI 정리 → 레일 덜컹임 → 셔터 역방향 상승 순서를 지킨다.
- 도감: 적/함정/보물/꽃/손패/조합/유물/용어 탭. 실제 데이터 정의를 요약 표시한다. 성수는 기본 사용 시 필드 랜덤 포자 2장 제거, 트리플 사용 시 필드 전체 포자 제거다.

- 상점 새로고침 버튼은 2줄 텍스트(`ReRoll`/`n$`)를 고정으로 사용한다. 새로고침 시 화폐 패널→버튼 블라스트, 버튼 임팩트, 유물 카드 1→2→3 순차 플립을 먼저 재생한 뒤 실제 리롤을 적용한다.
- 최신 UI 기준으로 리롤 플립은 제거되고, 유물 카드는 투명/소실 없이 즉시 교체 + 사각 버스트 이펙트로만 갱신 피드백을 준다.
- 카드팩 표기명은 `자원팩/강화팩/해금팩`, 하단 효과 문구는 `자원 보충/카드 강화/카드 해금`을 사용한다.
- 카드팩 타이틀/효과 문구는 `SHOP_PACK_LABELS`(ShopPools) 단일 데이터 소스를 사용해 상점 타일과 팩 선택 UI가 같은 라벨을 공유한다.
- 상점 리롤 플립은 **유물 3장만** 좌→우 순차로 한 바퀴(앞→뒷면→앞, `cardbackground_001.webp` 백페이스 플래시) 뒤집는다. 카드팩/무료카드는 리롤 애니메이션과 등급 테두리 대상이 아니다.
- 카드팩 구매 후 뜨는 3선택 카드는 플립/낙하 없이 제자리 페이드로만 표시한다. 선택 시에는 사각 블라스트만 재생한다.
- 무료 카드 슬롯은 카드백 플립 공개를 사용하지 않는다.
- 유물 리롤 플립에서는 카드 루트를 투명화하지 않는다. `shop-relic-flipper`만 회전시키고 front/back 2면을 항상 유지해, 회전 중 실제 카드 뒷면(`cardbackground_001.webp`) 노출 상태를 그대로 관찰/디버깅한다.
- 유물 리롤은 내부 face만이 아니라 카드 루트 전체(그림자/발광/테두리 포함)를 정확히 1회전시키고, 180° 구간에서 리롤 결과로 내용이 교체된다.
- 유물/카드팩 선택 플립 face(front/back)는 `translateZ` 오프셋 없이 같은 기준면(absolute + inset)에 둔다. 카드백 가시성은 z-index가 아니라 3D 회전/backface culling으로 결정되게 유지한다.
- 카드팩 개봉 3선택 카드는 하늘 낙하 대신 제자리 페이드 인 후, 유물 리롤과 같은 backface 플립(앞→뒷면 유지→앞)으로 공개한다.
- 상점 유물/무료/카드팩 개봉 선택 카드의 그림자·등급 발광·테두리는 카드 루트가 아니라 **앞/뒷면 face 레이어**가 소유해야 하며, 둥근 모서리 마스크도 flipper 레벨에서 클리핑해 플립 중 아트 돌출을 막는다.
- 플립 카드에서 카드백이 사라지면 front/back이 같은 기준면(absolute + inset)인지 먼저 확인한다. 한쪽만 normal-flow면 일부 GPU/브라우저에서 backface culling이 흔들려 카드백이 가려질 수 있다.
- 카드팩 개봉 3선택 카드는 카드백(뒷면) 비율 기준(세로형)으로 읽히게 유지한다. 텍스트 영역이 짧아도 카드 높이를 줄이지 않는다.
- 상점 셔터 이후 팩 선택용 신규 레이어는 레일 안쪽 상단에서 아래로 슬라이드해 내려오고, 이후 선택 카드가 순차 플립한다. EXIT는 팩 선택 중 숨긴다.
- 하단 카드팩 3슬롯을 받는 레이어는 완전 투명(배경/보더/글로우 없음)으로 유지해 일러스트만 읽히게 한다.
- 상점 새로고침은 유물 슬롯만 리롤한다(무료 카드/카드팩 고정). 리롤 시 이미 구매한 유물 슬롯은 빈칸으로 유지되고, 미구매 슬롯만 교체된다.
- 유물 리롤 플립은 손패 미리보기와 같은 2면 구조(`shop-relic-flipper` 안 front/back)로만 회전시키며, 카드 루트 레이어는 고정해 뒷면 투명/깨짐 현상을 방지한다.
- 상단 유물 레이어와 하단 카드팩 레이어는 버튼 위치를 유지한 채 전체 레이어만 소폭 좌측 보정한다.
- 카드팩 타일은 라인/테두리 스타일을 제거하고 일러스트+오버레이 중심으로만 표시한다.
- 상점 EXIT 카드 퇴장 연출에서는 이미 구매 완료(`is-purchased`) 상태 카드/타일을 제외해 재등장처럼 보이는 잔상을 막는다.
- 상점 팩 선택 오버레이는 하늘 낙하→카드백 플립 순서로 표시되고, 선택 카드 UI는 상단 이미지/하단 설명 카드 레이아웃을 사용한다. 모드 리스트는 최상단 z-index로 보장하며, 유물 리롤 플립은 카드백 노출 시간을 짧게 유지한다.

## UI/UX 규칙

- 기존 촛불/밀랍/낡은 카드 테마를 유지한다. 새 UI는 `GameBoardRenderer.ts`의 색/테두리/그림자/스크롤바 양식을 먼저 참고한다.
- 폰트는 `FontManager`와 OkDanDan을 사용하고 최소 12px 원칙을 깨지 않는다.
- 이모지 아이콘을 새로 넣지 않는다. 도감/검/하트/코인/방패/상점 보석처럼 **`src/ui/Icons.ts`의 플랫 inline-SVG path 아이콘**을 사용하거나 같은 방식(`currentColor`, 단색 fill/stroke, 작은 크기 가독성)으로 추가한다. 플레이어 카드의 방패 칩은 도감 런처와 같은 따뜻한 금색 계열을 따른다.
- 불빛/화폐/손패/체력/방패/불씨/게이지/피해/회복/처치 등 즉각 피드백은 출처 블라스트, 목적지 버스트, 수치/드롭 변화, 사망 축소가 분리되어 보이지 않게 같은 beat 안에서 처리한다. 꽃 관련 버스트는 공통 `flower-bloom`만 쓰지 말고 꽃 종류별 팔레트를 우선한다.

## 코드 작성 규칙

- TypeScript만 사용한다. import 주위에 try/catch를 두지 않는다.
- 새 코드에는 다음 작업자가 의도를 알 수 있는 짧은 주석을 남긴다. 단, 자명한 문법 설명은 피한다.
- 상태 변경은 가능한 한 시스템/엔티티에 두고, 렌더러는 표시/애니메이션 책임을 유지한다.
- 테스트용 시드, 더미 UI, 완료된 과거 실험 코드는 남기지 않는다. 발견 시 제거하거나 최종 보고에 명시한다.
