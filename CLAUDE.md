Unmelting 협업용 핵심 요약. **현재 코드 기준**으로만 유지한다.

## 프로젝트 요약
- 장르: 3레인 × 3행 카드 레인 로그라이트.
- 플레이: `녹지 않는 소녀` 1장 중심 + 손패/조합/유물.
- 톤: 따뜻한 촛불, 낡은 종이, 어두운 남보라/먹색 다크판타지.

- 무료 카드 `선물 상자`: 방문마다 등록된 n종 중 랜덤 효과 1개, 즉시 소모.
- 손패 `동전`은 보물상자 전용 보너스 드롭이며 일반 적 처치/드로우 풀에는 섞지 않는다.
- 키틴은 전방 선택 함정 1장을 제거한다. 단일은 2칸 이하, 트리플은 3칸 이하 폭(groupCount)까지 허용한다.
- 보스 현상금은 1~5$이며 1$ 단위 트레일/카운터로 증가하고, 전리품 유물 보상 트레일 목적지는 보유 유물 팬이다.
- 90F 조각사 소환 적은 사망 즉시 모델에서 제거한 뒤 보상 손패를 렌더해 재등장 잔상을 막는다.
- 보스/90F 규칙 타이틀은 제목 부상 완료 후 짧은 딜레이가 지나야 하단 클릭 문구가 점멸하고 입력이 열린다.
- 에나 수다 성향은 대사 읽음/스킵으로 런 안에서 즉시 `situationWeight`가 오르내리고, 경험 탭 수다 축은 이 단기 가중치를 반영하고, 수호 축은 실제 동작값을 유지한 채 표기만 0.75배로 완화한다. 런 종료 시 `adaptToOutcome()`이 단기 수다/예측 신호를 영구 성향에 소화해 저장한다. 예측/클러치가 현재 성향·예산 때문에 실제 발동하지 않았더라도, 후반부 고점 에나라면 개입했을 강적/보물 탐색 실패는 낮은 빈도로 '알고 있었지만 아직 못 도왔다'는 미숙 대사만 출력한다. 거미줄 미숙 대사는 단순 발견이 아니라 임박 병합 피해가 체력 절반 이상이거나 즉사 후보일 때만 출력한다. 소소한 클러치는 기본 확률에 clutchAdversityBoost·bondClimaxChance를 더해 위기/유대 상황에서만 일시적으로 상한을 연다.
- 에나 학습용 헤드리스 RL 시뮬레이터는 `src/rl/EnaTrainingSimulation.ts`에 있으며, "플레이어가 실제로 하는 게임을 에나 혼자서 미리 모험하며 준비"하는 그릇이다. UI/딜레이 없이 100층 한 호(arc)를 모델링한다: 3×3 전투 · 개별 손패(트리플 자동 합성) · 거미줄 병합(1/5/즉사)·포자 전염·씨앗 개화·이벤트 문 2턴 닫힘 · 불씨 티어 스폰 압박(실제 `EmberSystem` 버킷·`ENEMY_DEFINITIONS` 밴드 사용) · 10/20/30 상점·제단 + 유물 경제(방문마다 영구 성장) · 30/60/90/100F 실제 보스 스펙 · 90~100F 별빛 등반. 관측은 250차원(스칼라 34 + 9칸×14 + 손패 10×9), 행동은 21개 고정이다. 예지는 임박 거미줄/포자/강적을 우선하되, 강적 처치는 해금된 주요 공격 손패(불씨/모닥불/검과 방패/참격) 중 현재 공격력으로 즉시 처치 가능하고 과잉 피해가 가장 적은 카드를 고른다. 레시피/트리플 보조는 현재 체인·손패 순서에서 짧은 장수 안에 실제 발동 가능한 경우만 해금 손패 안에서 고른다. 반격 클러치도 시뮬에 포함한다.
- 에나 대사 사전은 LLM/API 없이 런타임에서 고르는 작성형 풀이다. 직업 선택, 모든 손패 획득/사용 전용·폴백, 주요 칸 이벤트/상호작용은 `CompanionLines.ts`의 고정문+역할 제한 템플릿 풀로 관리하고, `CompanionSystem`은 최근 선택·완성 문장 이력으로 반복을 줄인다. 손패 평은 실제 설명문 효과와 어긋나지 않게 쓴다(예: 방패 밀치기=방패 수치 피해, 이동/넉백 아님). 귀족 배경은 단정한 단어 선택으로만 남기고, 고풍 어미/긴 작대기 쉼표/클릭 멈춤 반응은 쓰지 않는다.
- 이벤트 NPC 대사는 레일 하단 말풍선(상단 꼬리)으로 출력하고, 클릭으로 타이핑 완료/다음 줄 넘김을 지원한다. 선택지는 대사 종료 후 긴 햄버거형 행 버튼으로 표시한다.
- 90F 보스 보상과 강제 시련이 끝나 셔터가 오른 직후, 남청~보라빛 `잿빛 굴레` 타이틀 연출과 함께 90~100층 별빛 규칙이 발동된다: `turnkey_001` 비합체 특수 보물 `별빛` 획득만 턴 +1, 다른 행동은 턴을 올리지 않는다.
- 별빛으로 100턴에 도달하면 즉시 클리어하지 않고 100F 보스 `녹지 않는 마녀`로 진입한다. 3×3 boss_004, 2턴 주기, 210/140/70 HP 페이지 경계 초과 피해 컷, HP바 경계선 표시, 1P 손패 소각·2P 4장 동시 전개 보스 손패(중복 시 추가 카드)·3P HP+10/ATK+3 강화 소환을 사용한다. 해금된 페이지 능력은 이후 페이지에도 유지되며 페이지 전환 이벤트는 최초 하향 돌파 1회만 발생한다.

## 스폰 가중치 표기 메커니즘 (`{{spawn}}`)

- 유물/시련 효과 문자열에서 `{{spawn}}` 토큰은 `GameBoardRenderer.relicEffectHtml()`이 실시간 ±% 수치로 치환한다.
- 기준값은 **항상 bright 티어** 고정 (`CardSpawner.getEffectiveWeightsForDisplay()`) — 불씨 게이지 변동은 표기에 영향을 주지 않는다.
- 영구 수정자(직업·유물·시련)만 반영: 유물 장착/시련 적용 시점에 % 표기가 갱신된다.
- 구현 위치: `src/systems/CardSpawner.ts` → `getEffectiveWeightsForDisplay()` / `src/ui/GameBoardRenderer.ts` → `relicEffectHtml()`

## 불씨 티어별 스폰 압박

- 티어는 절대 ember 수치 기준(bright ≥7, dim ≥4, flickering ≥1, extinguished =0)이며 emberMax와 무관하다.
- 티어 하강 시 **적·함정 가중치를 높이는** 방식으로 압박을 표현한다. 보물은 extinguished에서도 최소 5 유지.
- `CardSpawner`에서 `Math.max(1, ...)` 하한 보장으로 보물이 0이 되는 상황을 추가 방지한다.
