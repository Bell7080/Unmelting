# Unmelting

## 1. 프로젝트 개요

**Unmelting**은 “녹지 않는 소녀” 카드 한 장을 조작해, 매 턴 한 칸씩 다가오는 보상과 재앙의 카드를 판단하며 살아남는 **카드 레인 로그라이트** 게임이다.

플레이어는 일반적인 덱빌딩처럼 많은 카드를 손패로 운용하는 것이 아니라, **캐릭터 카드 한 장**을 중심으로 스테이지를 돌파한다. 스테이지 위의 카드들은 매 턴 플레이어를 향해 전진하며, 플레이어는 제한된 행동력 안에서 무엇을 처리하고 무엇을 감수할지 선택해야 한다.

---

## 2. 장르

**카드 레인 로그라이트 / 스테이지형 카드 생존 게임**

### 장르 정의

> 플레이어 캐릭터 카드 한 장을 성장시키며, 매 턴 한 칸씩 다가오는 카드판을 읽고 보상과 위협 사이에서 생존 루트를 만드는 로그라이트.

---

## 3. 한 줄 소개

> **녹지 않는 소녀 카드 한 장으로, 매 턴 한 칸씩 다가오는 보상과 재앙의 카드를 판단하며 살아남는 몽글몽글 다크판타지 로그라이트.**

---

## 4. 핵심 콘셉트

플레이어는 **“녹지 않는 소녀”**라는 캐릭터 카드 한 장을 조작한다.

스테이지에는 다음과 같은 카드들이 배치된다.

- 적 카드
- 보상 카드
- 난관 카드
- 저주 카드
- 이벤트 카드
- 상점 카드

이 카드들은 매 턴마다 플레이어 쪽으로 **한 칸씩 전진**한다.

플레이어는 제한된 행동력 안에서 다음을 판단해야 한다.

- 보상을 먹을지
- 적을 먼저 처리할지
- 함정을 피할지
- 위험을 감수하고 더 큰 보상을 노릴지
- 카드 위치를 조작해 위기를 넘길지

```text
아래 레인     [저주] → [빈칸] → [상점] → [소녀]
```

## 6. 기본 턴 구조

1. 플레이어가 행동력을 사용해 행동한다.
2. 모든 카드가 한 칸씩 플레이어 쪽으로 전진한다.
3. 근접 칸에 도달한 카드는 충돌 효과를 발동한다.
4. 빈 공간에 새로운 카드가 생성된다.
5. 다음 턴으로 넘어간다.

## 7. 핵심 재미

### 7.1 다가오는 카드판을 읽는 재미

카드들이 매 턴 전진하기 때문에 플레이어는 계속 판단해야 한다.

> “저 카드를 지금 처리할까, 한 턴 더 기다릴까?”

카드의 위치, 타입, 도달 타이밍이 곧 전투의 퍼즐이 된다.

---

### 7.2 보상과 위험 사이의 선택

이 게임에서 보상은 전투가 끝난 뒤 주어지는 것이 아니라, 위험 카드와 함께 전진해온다.

핵심 감정은 다음과 같다.

> “저 보상 먹고 싶은데, 지금 먹으면 위험하다.”

플레이어는 매 턴 안전한 선택과 욕심나는 선택 사이에서 고민하게 된다.

---

### 7.3 캐릭터 카드 한 장을 키우는 재미

덱을 크게 늘리는 방식이 아니라, **소녀 카드 한 장 자체가 점점 변화하고 성장**한다.

예시 성장 요소는 다음과 같다.

| 요소 | 역할                    |
| ---- | ----------------------- |
| 양초 | 생존 자원 / 회복 / 강화 |
| 우표 | 패시브 강화             |
| 밀랍 | 강하지만 위험한 강화    |
| 기억 | 스테이지 보상 능력      |
| 저주 | 리스크가 있는 능력 변화 |

플레이어가 느껴야 하는 감각은 다음과 같다.

> “이번 판의 내 소녀 카드는 전판과 다르다.”

---

### 7.4 카드 위치를 조작하는 재미

이 게임의 카드는 손패가 아니라 **공간 위의 오브젝트**다. 따라서 카드 위치 조작이 핵심 전략이 된다.

| 행동       | 효과                        |
| ---------- | --------------------------- |
| 밀치기     | 위험 카드를 뒤로 보낸다     |
| 끌어오기   | 먼 보상을 앞으로 당긴다     |
| 봉인하기   | 카드의 전진을 멈춘다        |
| 자리바꾸기 | 적과 보상의 위치를 교환한다 |
| 태우기     | 한 레인의 카드들을 제거한다 |

### 분위기 키워드

- 조용히 다가오는 재앙

## 9. 주인공

## 녹지 않는 소녀

양초로 이루어졌거나, 양초와 관련된 저주를 받은 존재. 평범한 양초처럼 녹아 사라져야 하지만, 이상하게도 그녀는 녹지 않는다.

그래서 세계는 그녀를 불길한 존재로 본다.

### 캐릭터 이미지

- 작은 소녀
- 촛불 또는 랜턴을 들고 있음
- 머리카락이나 옷자락에 밀랍 같은 질감
- 따뜻한 노란빛과 차가운 어둠의 대비
- 귀엽지만 무표정하거나 공허한 표정
- 종이 카드 위에 그려진 말 없는 인형 같은 인상

## 10. 카드 종류 (MVP: 3가지로 단순화)

| 카드 종류               | 역할                   | 특징                                              |
| ----------------------- | ---------------------- | ------------------------------------------------- |
| **적** (Enemy)          | 매 턴 플레이어를 타격  | 체력·공격력 보유, 제거 시 보상 드롭               |
| **함정** (Trap)         | 레인을 차단하는 장애물 | 선택하지 않으면 피하기 불가, 3칸 모두 함정 = 즉사 |
| **보물상자** (Treasure) | 강화/보상 카드 제공    | 매 턴 50% 사라지거나 10% 미믹(적)으로 변함        |

---

## 10.1 카드 그룹핑 메커닉

같은 레인의 같은 위치에 같은 종류 카드가 쌓이면 **합쳐져서 강화됨**:

### 적(Enemy) 그룹

- **2칸 차지**: 체력 +50%, 공격력 +1
- **3칸 차지**: 체력 +100%, 공격력 +2
- **예**: 잉크 늑대 2마리 = 체력 15 공격력 3인 슈퍼 늑대

### 함정(Trap) 그룹

- **1칸**: 일반 함정 (회피 필수)
- **2칸**: 받는 피해 +1
- **3칸 (모든 레인 막힘)**: 즉시 패배 (회피 불가능)

### 보물상자(Treasure) 그룹

- **2칸**: 보상 2배
- **3칸**: 보상 4배 (극히 드문 경우)
- 매 턴: 50% 확률로 사라짐, 10% 확률로 미믹(적)으로 변함

---

## 11. 턴 구조 (단순화된 MVP 버전)

```
1. 플레이어 선택 페이즈
   → 3개 레인 중 1개를 선택 (적/함정/보물상자 중 선택)

2. 플레이어 액션
   - 적 선택: 플레이어가 적을 공격 → 피해 입음
   - 함정 선택: 함정을 회피한 후 다음 카드가 진행됨
   - 보물상자 선택: 보상을 손패에 추가

3. 카드 진행
   → 모든 카드가 1칸 앞으로 이동 (플레이어를 향해 진행)

4. 도달 카드 처리
   - 적이 플레이어에 도달: 플레이어 받음
   - 보물상자가 플레이어에 도달: 매 턴 50% 사라지거나 10% 미믹 변환

5. 다음 턴 (1번으로 돌아감)
```

---

## 11.1 플레이어 액션 상세

| 선택              | 효과                 | 결과                             |
| ----------------- | -------------------- | -------------------------------- |
| **적 선택**       | 플레이어가 먼저 공격 | 적이 피해를 받고, 다음 턴에 반격 |
| **함정 선택**     | 함정을 피함          | 함정 제거, 그 자리 카드 진행     |
| **보물상자 선택** | 보상 획득            | 강화/보상 카드를 손패에 추가     |

## 12. 손패 시스템 (자원/아이템 기반)

덱빌딩이 아닌 **자원 재화** 느낌의 아이템 시스템:

### MVP 아이템 풀

| 아이템          | 효과              | 획득처  |
| --------------- | ----------------- | ------- |
| **체력 포션**   | 체력 +1 회복      | 적 처치 |
| **대형 포션**   | 체력 +2 회복      | 적 처치 |
| **공격 부스트** | 다음 턴 공격력 +1 | 적 처치 |
| **방어 부스트** | 다음 피해 -1      | 적 처치 |

### 12.1 손패 게이지와 카드 콤보 카운트

- 손패를 10장 사용할 때마다 선택한 게이지 보너스가 발동한다: 최대 체력 +5, 공격력 +1, 불씨 +3, 손패 랜덤 3장.
- 아이템 **카드**의 “손패 콤보 카운트” 효과는 이 10칸 게이지 진행도를 직접 올린다. 기본 카드는 +1, 트리플 카드는 +7이다.
- 카드 효과로 게이지가 10칸을 초과하면 보너스 발동 후 남은 수치가 다음 게이지 진행도로 이월된다.
- 레시피 조합은 실제 사용한 손패 카드만 재료로 본다. 카드 아이템이 올린 게이지 진행도는 레시피 재료 수를 대신하지 않는다.

### 추후 확장 아이템 (향후 계획)

- **함정 해제 키트**: 함정 제거 (자동 또는 선택)
- **열쇠**: 보물상자의 자물쇠 해제 (더 큰 보상)
- **서리 결정**: 적 동결 (1턴 공격 안 함)
- **시간 결정**: 턴 스킵 (카드 진행 안 함)
- **신비한 부적**: 특수 능력 활성화
- **저주 부적**: 양날의 검 아이템 (위험 vs 보상)

```
[UI 패널]                [게임 보드]
            상단 레인
    ▲      [카드] [카드] ●
    │      중앙 레인
  손  [카드] [카드] ⚔️ [플레이어]
```

- **카드 진행 방향**: 오른쪽에서 왼쪽으로 (플레이어를 향해)
- **좌측 UI**: 손패, 상태창
- **우측 UI**: 턴 정보, 선택 버튼

## 16. MVP 전투 규칙

```text
3레인 x 4칸
```

플레이어는 오른쪽 끝에 위치한다.

```text
A레인: [4칸] [3칸] [2칸] [1칸] [플레이어]
B레인: [4칸] [3칸] [2칸] [1칸] [플레이어]
```

1. 플레이어 행동 2회
2. 모든 카드 전진
3. 근접 카드 충돌 처리
4. 새 카드 생성

## 17. 포트폴리오용 설명문

> **Unmelting**은 캐릭터 카드 한 장을 조작해, 매 턴 한 칸씩 다가오는 카드들을 돌파하는 카드 레인 로그라이트입니다. 플레이어는 제한된 행동력 안에서 보상과 위협의 우선순위를 판단해야 하며, 적과 난관을 제거하거나 위치를 조작하면서 스테이지를 생존합니다. 게임의 핵심은 덱을 많이 늘리는 것이 아니라, “녹지 않는 소녀” 카드 한 장을 성장시키고 변화시키는 데 있습니다.

> **녹지 않는 소녀가, 한 칸씩 다가오는 재앙과 보상 사이에서 살아남는 카드 레인 로그라이트.**

### 핵심 재미 (MVP)

1. **다가오는 카드판을 읽는 재미**
2. **보상과 위험 사이에서 욕심내는 재미**
3. **매 턴 선택의 긴장감** (3개 선택지)
4. **자원 관리의 전술성** (손패 아이템)

### 분위기

> **따뜻한 촛불, 축축한 검은 숲, 귀엽지만 불길한 카드들, 그리고 녹지 않는 소녀.**

## 20. 손패 10칸 게이지와 UI 레이어 (MVP 업데이트)

### 손패 게이지 모드

손패 사용으로 채워지는 10칸 게이지는 좌측 모드 아이콘을 눌러 보상 모드를 순환 선택한다.

| 모드      | 10칸 충전 시 자동 발동 효과                  |
| --------- | -------------------------------------------- |
| 최대 체력 | 최대 체력 +5, 현재 체력도 같은 수치만큼 증가 |
| 공격력    | 공격력 +1                                    |
| 불씨 회복 | 불씨 +3 회복(최대치 초과 없음)               |
| 손패 획득 | 랜덤 손패 3장 획득(가득 찬 경우 초과분 손실) |

발동 순서는 동시에 처리하지 않고, 손패 카드 사용 후 조건을 만족하면 다음 순서를 유지한다.

```text
카드 효과 → 조합 효과 발동 → 게이지 효과 발동
```

게이지 효과는 체인 로그에 조합과 다른 강조 톤으로 기록한다. 10칸을 넘긴 진행도는 발동 후 다음 게이지 진행도로 남긴다.

### 손패 조합 카운트와 미리보기

- 손패 카드는 사용한 순간 체인에 실제 카드 1장으로 기록된다.
- `카드` 아이템은 별도 효과로 손패 콤보 카운트, 즉 10칸 손패 게이지 진행도를 추가한다. 따라서 일반 `카드` 사용은 실제 사용 1 + 카드 효과 1 = 게이지 2칸으로 기록한다. 트리플 `카드`는 실제 사용 1 + 트리플 효과 7 = 게이지 8칸으로 기록한다.
- 이 추가 게이지 진행도는 체인 배너에 중복 카드로 표시하지 않으며, 조합식 재료 판정에도 실제 손패 카드가 추가로 사용된 것으로 복제하지 않는다. 예를 들어 `셔플`은 반드시 `카드` 손패 2장을 실제로 사용해야 발동한다.
- 조합이 즉시 가능한 손패는 기존 반짝임에 더해, 카드 미리보기 옆에 발동될 조합식 이름과 간단한 효과를 함께 보여준다.

### 불씨 어둠 연출과 도감 조합 카드

- 불씨가 낮아질수록 전체 화면을 균등하게 덮기보다 좌우 사이드 어둠을 먼저 강화한다. 중앙 레일, 손패, 점수 정보는 어둡더라도 읽을 수 있어야 한다.
- 도감 조합 탭의 레시피 재료 미니카드는 하단 여백을 줄인 짧은 카드 비율을 사용한다.
- 도감 조합 미니카드가 호버로 펼쳐질 때는 도감 패널 경계에 잘리지 않고 바깥으로 나갈 수 있다. 카드 식별성이 패널 클리핑보다 우선이다.

### 플레이어 주변 UI 레이어

- 도감 버튼은 좌측 점수 패널에서 분리해 플레이어 카드 좌측의 투명 유틸리티 레이어에 배치한다.
- 플레이어 카드 우측에는 추후 유물 UI를 놓을 수 있는 투명 유물 레이어를 미리 확보한다.
- 체인 로그는 도감/유틸리티 레이어를 가리지 않는 위치와 톤을 유지한다.

### 도감 칸별 표기 양식

미믹, 함정, 보물상자처럼 같은 개체/계열의 폭만 달라지는 경우에는 별도 카드로 나누지 않고, 한 도감 카드 안에 `1칸 / 2칸 / 3칸` 효과 행을 함께 표기한다. 양초 생쥐/양초 개구리와 다른 추가 개체로 읽히는 `적 무리` 같은 항목만 별도 도감 카드로 분리한다.

| 계열     | 표기 방식                                                                                                   |
| -------- | ----------------------------------------------------------------------------------------------------------- |
| 일반 적  | 양초 생쥐/양초 개구리는 기본 능력치만 한 칸씩 표기하고, 2칸/3칸 강화 규칙은 별도 `적 무리` 카드에 모아 표기 |
| 미믹     | `미믹` 한 카드 안에 1칸/2칸/3칸 능력치와 드롭량을 행으로 표기                                               |
| 함정     | 같은 함정 한 카드 안에 1칸/2칸/3칸 피해 및 위협도를 행으로 표기                                             |
| 보물상자 | `보물상자` 한 카드 안에 1칸/2칸/3칸 드롭량과 변이 확률을 행으로 표기                                        |

### 레일 보충 규칙 (대형 제거 효과 대응)

`한 걸음씩`처럼 필드 전체 또는 한 줄을 크게 비우는 효과가 발동한 뒤에는 레일 정비가 한 번의 낙하/보충으로 끝나지 않는다. 각 레인은 아래칸부터 빈칸이 없어질 때까지 중력으로 당겨 내리고, 맨 위칸이 비는 순간에만 새 카드를 1장 뽑는다. 이 과정을 모든 칸이 연속으로 채워질 때까지 반복하므로, 랜덤 카드를 미리 묶음으로 뽑아두지 않는다.

## 21. 손패 조합식 레시피북 (MVP 업데이트)

조합식은 `src/data/Recipes.ts`의 데이터 테이블을 기준으로 관리한다. 추후 덱빌딩/해금 요소로 확장할 때는 레시피 추가·삭제를 이 테이블에서 먼저 처리하고, 새 효과 타입이 필요할 때만 `HandSystem`에 효과 구현을 추가한다.

| 조합식      | 재료                      | 효과                                               |
| ----------- | ------------------------- | -------------------------------------------------- |
| 따뜻함      | 양초 + 불씨               | 손패에 촛농 1장 획득                               |
| 점화        | 성냥 + 불씨               | 필드 모든 적에게 피해 1                            |
| 배당금      | 동전 + 동전 + 동전        | 1$                                                 |
| 셔플        | 카드 + 카드               | 랜덤 손패 1장 획득                                 |
| 양초 스매쉬 | 촛농 + 양초               | 전방의 랜덤 적 1장 처치                            |
| 지뢰제거반  | 키틴 + 성수               | 필드의 함정/저주/곰팡이 랜덤 1장을 보물상자로 변환 |
| 열쇠공      | 열쇠 + 열쇠               | 필드 랜덤 보물상자 1장 획득                        |
| 탐욕        | 열쇠 + 동전               | 대기라인의 보물상자가 아닌 1칸을 보물상자로 변환   |
| 한 걸음씩   | 촛농 + 양초 + 불씨 + 밀랍 | 필드의 모든 칸 제거                                |
| 도화선      | 성냥 + 성냥               | 전방 모든 적에게 피해 3                            |
| 성화        | 성수 + 불씨               | 전방 모든 칸 제거                                  |
| 밀매        | 밀랍 + 동전               | 대기칸 모든 보물상자 획득                          |
| 뜨거움      | 불씨 + 불씨               | 전방의 모든 적에게 피해 2                          |

현재 체인에서 어떤 손패를 지금 사용하면 조합식이 바로 발동되는 경우, 해당 손패 카드는 좌측 가장자리에서 바깥쪽으로 은은한 촛불빛이 퍼지는 힌트를 표시한다. 이 힌트는 실제 발동 판정과 같은 레시피 테이블을 미리보기로 사용하므로 조합식 커스터마이징과 함께 자동으로 갱신되어야 한다.

## 상점/유물 1차 구현 메모 (2026-05-13)

### 상점 진입

- 10턴을 완료할 때마다 레일이 `쿠궁!` 흔들린 뒤 3×3 레일 칸을 덮는 셔터가 내려오고, 셔터가 다시 열리며 상점 오버레이가 등장한다.
- 상점은 현재 유물 전용 임시 탭으로 구성한다. 내부 물품 확장은 이후 상점 아이템 카탈로그를 추가해 같은 오버레이에 붙인다.
- 상점 재화는 `$`, 최대체력, 공격력 비용을 모두 지원한다. `$`가 아닌 비용은 캐릭터가 최소 1의 해당 능력치를 유지할 수 있을 때만 구매 버튼이 활성화된다.
- 진열은 매 방문마다 보유 중이 아니고 영구 제거 목록에도 없는 유물 중 랜덤 3개다.

### 유물 UI

- 플레이어 카드 오른쪽의 투명 유물 레이어가 실제 보유 유물 카드 프리뷰 영역으로 전환됐다.
- 유물 카드는 `일러스트 → 제목 → 효과` 구조를 따른다. 현재 일러스트는 임시로 양초 생쥐 이미지를 사용하며, 추후 유물 전용 webp로 교체한다.

### 이펙트 타이밍 조정

- 함정을 밟을 때 피해 숫자/피해 버스트와 함정 소모 이펙트가 순차적으로 `타/닥` 분리되지 않도록 같은 비트에서 함께 시작한다.
- 적의 플레이어 피해도 숫자와 버스트가 한 번의 피해 임팩트로 묶이도록 조정한다.

### 유물 도감/로그/보관 UI 보강 (2026-05-14)

- 도감에는 `유물` 탭이 추가되어 현재 상점 유물의 효과, 비용, 보유 여부를 한 곳에서 확인한다.
- 도감 조합 레시피 미니카드는 도감 본문 스크롤을 유지하기 위해 실제 카드 영역은 스크롤 컨테이너 안에 두고, 호버 중인 재료 스택만 body 고정 레이어로 복제해 패널 밖에서도 보이게 한다.
- 유물 발동은 활동 로그에서 `유물` 배지로 분리하고, 체인 배너 아래 작은 토스트형 텍스트로 함께 표시한다.
- 플레이어 카드 오른쪽 유물 보관 레이어는 플레이어 카드 높이에 맞춰 세로 스크롤/줄바꿈 그리드로 표시하며, 슬라이더는 도감/로그와 같은 촛불색 스크롤바를 사용한다.
