# Unmelting Game Concept (2026-05-21 최신화)

## 1) 게임 정체성
- 장르: 카드 레인 로그라이트.
- 코어: 3레인 × 3행 필드에서 카드가 턴마다 전진하며, 플레이어는 `녹지 않는 소녀` 1장과 손패/조합/유물로 생존 루트를 만든다.
- 감정: 안전/탐욕 사이의 즉시 의사결정.
- 미감: 따뜻한 촛불 + 낡은 종이 + 어두운 남보라/먹색의 몽글몽글 다크판타지.

## 2) 현재 플레이 루프 (구현 기준)
1. 플레이어가 보드/손패 행동을 수행한다.
2. 적/보물/폭탄/꽃 등 턴 요소가 처리된다.
3. 불씨 감소, 레일 하강/리필, 그룹화가 적용된다.
4. 포자 카운트가 2→1→0으로 표기된 뒤 전염이 적용되고, 전염 후 2턴으로 리셋되며 즉시 그룹화가 보정된다.
5. 10턴마다 상점 단계로 진입한다.

> 참고: 상점/제단 구매 및 EXIT 자체는 일반 턴·포자 타이머를 진행시키지 않는다.

## 3) 필드/오브젝트 규칙
- 기본 필드: 3레인 × 3행.
- 전방 행 중심 인터랙션, 전방 동일 카드 그룹화.
- 꽃은 시작 3×3에 직접 등장하지 않고 씨앗으로 대기 라인에서 전방 도달 시 발화.
- 굳음(봉인) 상태의 턴 타이머형 요소(폭탄/포자/꽃/보물)는 해동 전까지 진행 정지. UI는 중앙 텍스트(굳음/n턴)만 띄우고 하단 HP바 영역은 투명 그라데이션으로 남긴다. 해제 시 0턴을 짧게 보여준 뒤 해동된다.

## 4) 손패/조합/게이지
- 시작 손패 없음, 드롭 기반 획득.
- 자동 3장 병합 + 조합(레시피) 발동.
- 키틴: 전방 선택 함정 1장 제거. 일반은 2칸 이하, 트리플은 3칸 이하 폭의 함정까지 제거한다.
- 콤보 게이지(촛불) 보상은 외부 보상으로 10 이상 도달해도 즉시 처리.
- 획득/회복/증가 피드백은 트레일·버스트·숫자 롤링을 같은 beat로 동기화.
- **공격 손패 피해 공식**: 불씨 단일 = 1.0공+1, 트리플 = 3.0공+5(공 = 플레이어 공격력). 강화팩 보너스는 별도 합산. 도감/손패 미리보기 기본 표기는 합산 수치, Shift 누름 시 수식(공 아이콘+상수)으로 전환.
- **Shift 자세히보기**: 손패 미리보기·도감에서 Shift 키를 누르는 동안 피해 수치가 수식으로 전환된다.

## 5) 상점/제단 시스템
### 5-1. 상점 진입 주기
- 10턴: 상점
- 20턴: 상점
- 30턴: 제단(상점 변형)
- 이후 같은 패턴 반복(100턴 러닝 토대)

### 5-2. 상점(일반)
- 구성: 리롤 + 유물 3 + 무료 카드 1 + 카드팩 3.
- 리롤: 유물 슬롯만 교체(무료/팩 고정), 비용은 방문 내 사용 횟수에 따라 증가.
- 카드팩: 일반 상점 3팩(자원/조합/해금)은 공통 시작가 — 10층 120에서 10층마다 +40 (20F 160, 40F 240 …). 같은 방문에서 동일 팩을 열 때마다 시작가만큼 증가한다(예: 120→240→360). UI 표기와 실제 차감은 공통 가격 함수 기준.

### 5-3. 제단(30/60/90턴)
- 구성: 유물 3 + 무료 카드 2(기본/동전 한 닢) + 카드팩 4(축복/자원/강화/삭제).
- 리롤 없음.
- 팩 가격: 30F 500, 60F 1500, 90F 2500 시작. 같은 팩을 열 때마다 시작가만큼 증가한다(예: 1500→3000→4500).

### 5-4. 시각/타이밍 원칙
- 셔터/veil 이후 상호작용 UI는 `shop-content-bundle`로 한 번에 공개.
- 시작 직업 선택은 전체 화면 모달이 아니라 빈 레일 내부 암막/커튼 레이어에서 보여주며, 직업 확정 후 닫힌 커튼 뒤에서 최초 3×3 보드를 채운 다음 공개한다.
- 이벤트 문은 레일 커튼이 천천히 닫힌 뒤 일러스트를 세로 슬릿에서 좌우로 열어 보여주고, 대사가 끝난 뒤에만 긴 햄버거형 선택 행을 공개한다. 종료는 선택지/일러스트/커튼의 역순으로 닫아 입력 복귀 시점을 명확히 한다.
- 이벤트는 choice형(대사→선택지)과 minigame형(대사→인터랙티브 미니게임) 두 종류다. 공용 진입 셸(`GameBoardRenderer.openEventScene`)을 공유한다. 미니게임은 **정산 지연 없이 실시간**으로 진행한다: 각 액션마다 `EventResourceSink`로 실제 자원(불빛/체력/게이지/방패/손패)을 즉시 올리고 내리며 기존 HUD 카운터 피드백 + 획득 트레일/버스트를 그대로 쏜다(메타 화폐 $ 는 이벤트에서 안 쓴다). 패널은 테두리/박스 배경 없이 은은한 방사형 어둠만 깔아 뒷배경 일러스트와 경계를 허물고(자원 수치는 실제 HUD가 보여줌), 종료 시 패널을 접은(`is-closing`) **뒤에** 이벤트1처럼 마무리 대사(`outro`)를 출력하고 커튼을 연다.
  - `event_002` 겁쟁이 미니언 아슬아슬 흥정: 실력=위험 관리(로스트아크 돌깎기식). 정해진 기회(`attempts`) 안에서 각 시도는 **가시 성공 확률**로 판정한다. **성공할수록 불안↑ → 성공 확률↓, 실패하면 불안↓ → 확률 회복**. 옵션 두 부류 — 탐욕형(aim success, 성공=보상/실패=디메리트)은 확률 높을 때, 협박(aim fail, **실패=대박/성공=디메리트**)은 확률 낮을 때 굴려 갈아탄다. 노린 결과의 불빛엔 불안 비례 `riskPremium`이 붙는다. 불안 pip은 별빛 다이아를 호박~적색으로 발광.
  - `event_003` 가위바위보 백작: 벅샷 룰렛식 아이템 도박. 백작 덱은 **조성(개수)만 공개·순서는 섞여 숨김**(탄창)이라 카운팅으로 확률을 읽고, 다양한 자원으로 산 아이템(엿보기=불빛/흩기=게이지/갑절=체력/부적=방패/미끼=손패)으로 다음 패를 들추거나 판돈·손실을 조작한다. **비김도 판돈의 `tieLossFraction`(기본 50%) 레이크**가 있어 무손해 손이 없다. 불빛을 걸고 연승 배율을 쌓으며 언제든 물러나고, 덱을 비우고 순이익이 `relicWinMultiple×판돈` 이상이면 유물 보상. UI는 이벤트1처럼 테두리 없는 폰트/이미지 위주 — 손패는 정사각 둥근모서리 풀인 마스크 이미지 타일(`rps_rock/paper/scissors.webp`, 없으면 텍스트 폴백), 아이템은 부채꼴, 판돈은 별 개수+수치, 백작이 낸 패는 상단 얼굴 부근 슬롯에서 슬롯머신처럼 돌다가 착지한다.
- 팩 등장 타이밍은 슬롯 인덱스 기반 스태거(`--shop-pack-order`)로 제어하여 3슬롯/4슬롯 모두 동일 규칙 유지.
- 리롤 버튼 노출 타이밍은 카드 enter beat와 동기화.

## 6) 무료 카드/보상 규칙
- `무료 카드(free_001)`: 상점/제단 공통 타이틀. 방문마다 등록된 n종 랜덤 효과 1개(불빛/화폐/체력/게이지/손패) 후 즉시 소모.
- 제단 `동전 한 닢(free_002)`: 제단 방문마다 다시 활성화되며, 화폐 패널 블라스트 후 1$ 롤링 증가.
- 제단 `자원팩`: 두꺼운 의복(최대체력 +5), 가열(공격력 +1), 큰 배낭(최대 손패 +2), 성냥갑(최대 불씨 +2), 두꺼운 심지(불씨 소모 주기 +1턴), 조커 카드(콤보 게이지 -1), 랜턴(불빛 +10%), 동전 한 닢(1$) 중 weight 기반 3장을 제시한다.
- 무료 보상 블라스트 목적지는 보상 타입별 HUD로 분기.
- 보스 현상금은 1~3$이며 1$ 단위 트레일/카운터로 증가하고, 전리품 유물 보상은 획득 가능 유물이 없어도 불빛 패널이 아니라 보유 유물 팬을 목적지로 사용한다.
## 7) 희귀도/데이터 소스
- 손패 `동전`은 보물상자 전용 보너스 드롭이다. 보물상자는 일반 드롭 풀에 동전만 추가로 섞는다.
- 희귀도 팔레트: common / rare / epic / unique / legendary.
- 유물/팩/선택 UI의 희귀도 표시는 공통 소스(`src/data/ShopPools.ts`)를 사용한다. 상점/제단 유물은 등급별 발광으로 희귀도를 읽게 한다.
- 유니크 등급은 예외 메커니즘(예: 희망) 중심으로 제한 사용.

## 8) 보스/시련 상태
- 30/60/90턴 제단 EXIT 후 보스 게이트로 진입한다. 100턴은 별빛 최종 등반 뒤 즉시 최종 보스 게이트로 진입한다.
- 30F `양초 백작`: 3×3 보스, 일정 HP 손실마다 랜덤 손패 지급. 보스 공통으로 밀랍 굳음이 적용되며, 굳은 동안 도래한 반격/특수행동 beat를 건너뛴다.
- 60F `불씨 기사단장`: 숨은 정체는 레온하르트이며 플레이어 응답 대사로만 암시. 3×3 보스, boss_002 일러스트, 좌→우 느린 돌입 후 쿵 착지, `저택의 방패` 타이틀, 3턴마다 기사단장의 손패 2장 발동. 획득한 밀랍 방패는 플레이어처럼 HP 게이지 좌상단의 큰 방패 아이콘 안에 수치로 표시한다.
- 90F `밀랍 조각사`: 2×3 보스, 3턴마다 후방 이동/소환 페이즈를 사용하며 소환 적 사망 시 재렌더 잔상 없이 즉시 필드에서 제거한다.
- 보스 격파 후 레일형 3보상 드롭을 모두 획득하고, 강제 시련 선택 후 일반 턴으로 복귀한다.
- 90F `밀랍 조각사` 격파 후 보상과 강제 시련이 끝나 셔터가 오른 직후, 짙은 남청~보라빛 타이틀 연출(`잿빛 굴레 / - 별빛으로 굴레를 벗어나리 -`)과 함께 90~100층 전용 규칙이 발동한다. 이 구간에서는 적/함정/일반 보물/꽃 행동으로 턴이 오르지 않고, `turnkey_001` 일러스트의 비합체 특수 보물칸 `별빛`을 획득할 때만 턴이 +1 오른다. 별빛 10개를 모아 100턴에 도달해야 최종 보스 구간으로 넘어간다.
- 100F `녹지 않는 마녀`: `잿빛 굴레의 주인`, boss_004 일러스트, 양초 백작과 같은 3×3 크기, HP 210/ATK 15/2턴 주기. 140/70 경계는 초과 피해를 버리고 전환 대사 후 다음 페이지로 넘어간다. 1페이지(210~141)는 HP 10 손실마다 랜덤 손패 2장을 소각한다. 2페이지(140~71)는 공격 전 보스 손패 4장(방패5/체력5/피해5)을 사용하고 같은 카드 2장 이상이면 해당 효과를 추가 1회 발동한다. 3페이지(70~0)는 3×2 후방 대기로 물러나 강화 소환수(HP+5/ATK+3)를 불러내며, 전멸 시 전방 복귀와 공격 주기 초기화 후 다시 반복한다.

## 9) 경제 밸런스
- 불빛 획득 턴 보정은 완만한 선형 증가(1 + 턴×0.015)를 기준으로 하며, 90턴대 과도한 6자리 누적을 피한다.

## 9-1) 에나 RL/딥러닝 학습 시뮬레이션
- 구현 위치: `src/rl/EnaTrainingSimulation.ts`.
- 목적: 실제 렌더링·애니메이션·턴 딜레이 없이 3×3 보드, HP/방패/불씨/손패성 자원, 적/함정/보상/꽃 위협, 손패 위치 사용, 상점/이벤트 선택, 30층 보스 준비와 실제 데이터 정의 기반 손패·유물·시련·직업·이벤트 연계를 빠르게 반복 학습한다.
- 출력: 272차원 관측 벡터(스칼라 38 + 예고 3칸×6 + 9칸×14 + 손패 10×9) + 25개 고정 행동 인덱스(레인 3 + 손패 10 + 대기 1 + 상점 9 + 이벤트 2) + `(state, action, reward, nextState, done)` 전이 샘플 + `analyzeDecision()` 전략 문장 + `EnaKnowledgeAdapter` 자동 전술 리포트 + `EnaEffectProbe` 실제 HandSystem 검증/플레이 로그 튜닝 + `EnaRuntimeObserver` 실시간 런 관측/시각 프레임. 외부 DQN/PPO 학습기는 `collectDataset()` 또는 `EnaPolicy` 교체로 연결하며, 학습된 순수 TS 정책망은 `EnaPolicyStore`가 feature/action 계약을 검증한 뒤 저장·복원한다. 실제 런 결과는 `EnaAutonomousLearner`가 플레이어에게 보이지 않는 내부 자기반성/선호 신호로 압축해 다음 판단 재료로 보관하고, 새 런 시작 때 낮은 빈도로 자연스러운 기억 대사로 표현한다.
- 상점/제단 국면은 실제 콘텐츠와 같은 행동으로 학습한다: 무료카드(선물 상자 6종/제단 수당 3$), 유물(상점 불빛 구매·제단 무료 1픽), 일반 상점 3팩(자원/조합/해금)·제단 3팩(확률/삭제/자원), `ShopPricing` 공유 가격 + 방문 내 반복 누적가. 재화는 불빛(팩·유물)과 화폐(보물 동전·보스 현상금)로 분리되어 있다.
- 기본 교사 정책은 클러치 생존(치명 함정/저체력 방패), 불씨 부족 시 성냥 보급, 예측 대비(전방 위협 우선 손패 보급), 손패 보존/사용 위치, 트리플 대기 가치, 레시피/유물 시너지, 상황별 상점 선택(위급 시 자원팩, 잠긴 카드 다수 시 해금팩, 제단 자원팩 우선), 플레이 로그 보정, 보스 버스트 준비, 2~3수 뒤 레일 위험을 함께 판단한다. 교사 정책 임계값은 `DEFAULT_ENA_HEURISTIC_POLICY_CONFIG`로 분리해 실험별 오버라이드가 가능하다.
- 에나의 예측 손패 보급은 건넨 손패가 즉시/위기 타이밍에 쓰였는지, 청소 손패가 실제로 거미줄·함정을 얼마나 제거했는지를 `predictiveWeight`에 반영한다. 경험 탭 수호 축은 실제 동작값을 바꾸지 않고 표기만 0.5배로 완화한다.
- 지원 카드 선택은 `HandCardAdvisor` 범용 스코어러가 담당한다 — 새 손패/유물은 데이터에 `synergyTags`(공격 카드는 `damageProfile`)만 달면 에나 판단·지식·학습에 자동 반영된다.
- 에나는 런 내 기분(mood)과 런을 넘는 유대(bond)를 유지하며, 유대·구조화 런 기억(도달 층/사망 원인)은 자기학습 저장에 함께 영속되어 새 런 회상과 보스/종막/시련/팩/별빛 전용 대사의 재료가 된다.

## 10) UI 스타일 가이드 (실무 요약)
- 새 UI는 기존 촛불/밀랍/낡은 카드 테마의 색감·그림자·질감 규칙을 따른다.
- 아이콘은 이모지 대신 `Icons.ts` 단색 SVG 계열 사용.
- 카드/버튼/패널의 애니메이션은 “동시에 읽히는 beat”를 우선한다.

## 11) 문서 운영 원칙
- 이 문서는 **현재 사실** 중심으로 유지한다.
- 미구현 장기 아이디어는 별도 백로그로 관리하고 본문에 플레이스홀더를 늘리지 않는다.

## 12) 거점(촛대) 화면 — 진입 허브 / 9칸 스테이션

> `/시작` 진입점. 구현: `src/ui/hearth/HearthScene.ts` · 스타일 `HearthStyles.ts`.
> 기본 부팅은 여전히 직행 인게임(`startGame()`)이고, 거점은 `/시작`에서만 깨어난다(인게임 테스트 편의).

### 12-1. 콘셉트
- 타이틀 = 로비 = 메타상점이 **한 화면**. 별도 씬 컷 없음.
- **인게임 빈 3×3 레일을 배경으로 재사용**한다(셰어드 무대). 거점은 레일 위 오버레이 레이어.
- 진입 연출: 거대한 문을 열듯 **칙칙한 오로라빛 커튼**(직업 선택 `job-rail-curtain` 재사용·hue-rotate retint)이 갈라지며, 폐저택/영지풍 배경이 드러나고, 하단 중앙 `모험` 칸이 **화륵** 점등된다.
- 칸 디자인/전환은 **상점 셔터·이벤트 커튼 코드를 재활용**한다(모험 클릭 → `playShopTransition()` 셔터 하강 → 모험 설정 페이지).
- 설정/도감은 칸이 아니라 **플레이어 카드 옆 아이콘**으로 뺀다.
- **메타 화폐(굳은 밀랍)는 런에서만 나온다. 방치(시간 기반) 적립 없음.** 로비의 밀랍 드립은 순수 앰비언트.

### 12-2. 9칸 배치 (잠정 좌표 — 게임플레이에 따라 재배치 가능)
하단 행 = 전방(런 직전 준비), 중앙·상단은 진행도 해금으로 점등.

```
상단:  [암시장]   [타로]        [도박장]
중단:  [길드]     [잿빛 굴레]   [서고]
하단:  [무역]     [모험]        [만찬]
```

`HearthScene.renderCells()` 인덱스(0~8, row-major): 0 암시장 / 1 타로 / 2 도박장 / 3 길드 / **4 잿빛 굴레(중앙)** / 5 서고 / 6 무역 / **7 모험** / 8 만찬.

### 12-3. 칸별 역할
| 칸 | 역할 |
|---|---|
| **모험** | 런 시작. 클릭 시 셔터 하강 → 캐릭터 선택(하단 얇은 프로필 카드 스트립) → 선택 블라스트가 플레이어 카드에 꽂힌 뒤 출발 버튼 공개. 난이도/챌린지는 추후 모험 설정 페이지로 확장한다. |
| **무역** | 메타 제단의 기본 축. 손패 개방·**유물 잠금 시스템**·**다음 판 계승 시스템**·추가 효과 등을 메타 화폐로 영구 해금. 현재 임시 UI는 암전/블러 없는 `hearth_bg_004` 셔터 배경, 색/테두리 없는 좌측 중앙 정렬 텍스트 라벨 탭(1번~6번), 우측 탭별 중앙 정렬 빈 카드팩 5개 가로 슬라이드 레일과 하단 타원 그림자로 구성한다. |
| **만찬** | 모험 직전의 일회성 버프("최후의 만찬"). 유물 인벤토리에 만찬 유물이 없을 때 **무료 만찬 1회**를 제공한다. 현재 임시 UI는 아주 약한 암전/블러만 얹은 `hearth_bg_005` 배경, 중앙 가로 카드팩 레일, 무료 팩 선택 후 음식 1회 + 추가 스탯 2회 3단계 선택, 완성 음식의 실제 유물 `최후의 만찬(last-supper)` 인벤토리 저장, 선택한 음식·소스·토핑으로 카드 이름/효과/하단 설명을 생성하고 인게임 시작 시 각 스탯을 1회 적용, 이후 인벤토리 내 만찬 카드 기준 `hearth_006` 일러스트와 고정 대사 재방문 화면으로 구성한다. |
| **길드** | 업적 관리. 업적으로 **캐릭터 등 해금**도 연결. |
| **암시장** | **턴 갱신마다 바뀌는 정해진 품목**을 메타 화폐로 구매. |
| **타로** | 운명 점치기. 용도 미정(분위기와 대비되는 밝고 예쁜 일러스트로 톤 환기). |
| **서고** | 기록보관소. 전적/업적/로어를 보관하고, **그 기록을 바탕으로 영구 추가 효과(패시브)** 를 부여한다. (옛 `현상금` 칸 자리. 퀘스트는 좌측 패널로 이관) |
| **잿빛 굴레** | 엔드리스 모드. **200층 진엔딩 클리어 시 개방.** |
| **도박장** | 메타 코인을 소모해 유희/재화 증식. |

> 길드 vs 서고 구분: **길드 = 업적으로 콘텐츠 해금(캐릭터 등)**, **서고 = 기록 기반 게임플레이 버프**. 역할이 겹치지 않게 유지한다.

### 12-4. 점등(해금) 게이팅
- 현재 빌드는 확인/테스트 편의를 위해 **9칸을 모두 임시 해방**한다. 모험만 셔터/캐릭터 선택/출발 동작을 갖고, 나머지는 점등 애니메이션과 인스펙터 확인용 칸이다.
- 추후 게이팅을 되돌릴 경우에도 꺼진 칸은 제거하지 말고 **다음 해금 목표를 어렴풋이 비추는 어둠**으로 남긴다.

### 12-5. 거점 패널/슬롯 (런 UI 슬롯을 메타 용도로 치환)
거점에서는 보드 레이어를 살려두되 런 전용 내용물만 메타 용도로 갈아끼운다(`body.hearth-lobby`).
- **화폐 슬롯**: 맥락에 따라 다른 지갑 — 거점은 **메타 재화(굳은 밀랍)**, 런은 코인($). 거점↔런 연속성 앵커.
- **좌측 패널** = **퀘스트(옛 현상금 역할).** 의뢰를 **딱지(기차표/라벨) 목록**으로 노출. 딱지엔 **등급명 + 진행 카운터만**(사소한 정보), 세부(목표/보상/기한)는 hover 시 우측 인스펙터에 표시.
  - 등급 3종, **색으로 중요도**(카테고리 헤더 없음): **중요한 의뢰**=밀랍 황금빛(`--color-flame`, 발광/엠보싱, 예 `0/100`) → **적당한 의뢰**=바랜 청동(예 `0/10`) → **사소한 의뢰**=탁한 회색(무광/반투명, 예 `0/1`).
  - 색만 의존하지 않도록 **순서(중요한 게 상단 핀) + 발광/엠보싱** 두 단서만 보강(텍스처·씰 제외).
  - 의뢰가 많아지면 **화살표로 접고, 누르면 펼친다.**
  - **보상은 미정 — 지금은 아무것도 지급하지 않는다(구조만).** 완료분은 서고(기록)로 아카이브.
- **우측 패널** = **인스펙터(정보창).** 평소 비움(여백의 미), 칸/딱지 hover 시 상세가 스르륵 부상하고 떼면 사라진다.
- **하단 플레이어 카드** = 로비 진입 시 런 전용 UI와 함께 비워 둔 상태로 보이고, 모험 셔터 안에서 캐릭터를 선택하면 빛 카드가 꽂히며 선택 캐릭터 프로필로 갱신된다. 현재 해강 외 캐릭터는 추후 설계 연동용 임시 슬롯이다.
- **하단 우측 유물 인벤** = **계승 유물 미리보기**(무역의 다음 판 계승 시스템과 연동). 계승 유물을 얻거나 채웠을 때만 표시하고, **없을 땐 '없음' 딱지 없이 그냥 비운다.**
- **거점→런 패널 전환(출발 직후):** 좌측(퀘스트)·우측(인스펙터) 패널은 출발을 누르는 즉시 **각각 좌/우 바깥으로 슬라이드 아웃**한다. 직업 선택창이 열리는 타이밍에 맞춰 **런 버전(좌=턴/불빛/로그, 우=손패/콤보게이지/확률)이 슬라이드 인**으로 교체되어 들어온다. **화폐 슬롯만 같은 위치를 공유하므로 슬라이드 없이 그대로 유지**(밀랍→코인 내용만 전환).
